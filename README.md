# develop_agent — AI Development Operating System

AIエージェントを中核に据えた、プロダクト開発のためのオペレーティングシステム（開発OS）。
企画からグロースまでの全工程を、構造化されたディレクトリとAIエージェントの協働によって推進する。

---

## 目的

- **開発プロセス全体の体系化**: 企画・UX・UI・AI・実装・テスト・リリース・グロースまで、プロダクト開発のライフサイクル全体をひとつのリポジトリで管理する。
- **AIエージェントによる開発の加速**: 各工程に特化したAIエージェントを配置し、人間は意思決定に集中、定型作業や成果物の生成はAIに委譲する。
- **知識の資産化**: プロンプト・テンプレート・ドキュメントを再利用可能な形で蓄積し、プロジェクトを重ねるごとに開発OSそのものが進化する仕組みをつくる。

## 思想

1. **プロセスをコードのように管理する**
   開発の流れ・判断基準・成果物のフォーマットをすべてリポジトリ上に明文化し、バージョン管理する。暗黙知を残さない。

2. **工程ごとの関心の分離**
   `00_System` 〜 `08_Growth` の番号付きディレクトリが開発ライフサイクルの各フェーズを表す。各フェーズの成果物は対応するディレクトリに置き、フェーズ間の依存を明確にする。

3. **人間は意思決定、AIは実行**
   AIエージェントはドキュメント生成・コード実装・テストなどの実行を担い、人間はレビューと意思決定に集中する。最終判断は常に人間が行う。

4. **再利用ファースト**
   一度作ったプロンプト・テンプレート・事例は `prompts/` `templates/` `examples/` に汎化して保存し、次のプロジェクトで即座に使えるようにする。

## ディレクトリ構成

```
develop_agent/
├── README.md          # 本ファイル
├── .gitignore
├── docs/              # 開発OS自体のドキュメント（運用ルール・設計思想など）
├── templates/         # 各工程で使う成果物テンプレート
├── prompts/           # AIエージェント用の再利用可能プロンプト
├── examples/          # 成果物のサンプル・過去事例
├── agents/            # AIエージェントの定義（今後追加）
│
├── 00_System/         # 開発OS自体の設定・共通ルール・ワークフロー定義
├── 01_Product/        # プロダクト企画・要件定義・ロードマップ
├── 02_UX/             # ユーザー調査・ペルソナ・ユーザーフロー・情報設計
├── 03_UI/             # デザインシステム・画面設計・プロトタイプ
├── 04_AI/             # AI機能の設計・モデル選定・プロンプト設計・評価
├── 05_Development/    # 実装・アーキテクチャ・技術選定・コーディング規約
├── 06_Test/           # テスト計画・テストケース・QA・品質基準
├── 07_Launch/         # リリース計画・デプロイ・運用手順・監視
└── 08_Growth/         # 分析・改善施策・マーケティング・グロース戦略
```

## 運用方法

### 基本フロー

1. **フェーズに沿って進める**: `01_Product` から順に成果物を作成し、各フェーズの完了条件を満たしてから次へ進む。
2. **成果物は必ずリポジトリに残す**: 議論の結論・決定事項・仕様はすべて対応するディレクトリ内のMarkdownファイルとして記録する。
3. **テンプレートから始める**: 新しい成果物を作るときは `templates/` を確認し、あればそれをコピーして使う。なければ作成後にテンプレート化を検討する。
4. **AIエージェントの活用**: 各工程で `prompts/` のプロンプトや `agents/` のエージェント定義を使ってAIに作業を委譲する（エージェントは今後整備）。

### 運用ルール

- **命名**: ファイル名は内容がわかる英語のケバブケース（例: `user-persona.md`）を基本とする。
- **1ファイル1トピック**: ドキュメントは肥大化させず、トピックごとにファイルを分ける。
- **更新はコミットで追跡**: 重要な意思決定の変更は、コミットメッセージに理由を残す。
- **フェーズ横断の共通事項**は `00_System/` に置く。
