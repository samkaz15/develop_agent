## 1. コンテンツ戦略総論

### 1.1 基本命題

> **コンテンツは広告の代替ではなく、プロダクトの延長である。占いコンテンツ自体が「製品サンプル」だ。**

- 「今日の乙女座の運勢」を読む体験は、本サービスの鑑定体験の**縮小版そのもの**である。コンテンツの言葉の質＝プロダクトの言葉の質としてユーザーに知覚される
- したがって、コンテンツの品質基準はマーケティング基準（CTR・PV）ではなく**プロダクト基準**（読んだ人の心が軽くなるか＝WRSの定義と同型）で判定する
- 判定文: **「このコンテンツを読み終えた人が、保存・シェア・『自分も占ってみる』のいずれかをしたくなるか？」に編集会議でYesと言えないものは公開しない**

### 1.2 コンテンツが担う3つの機能

| 機能 | 対応ループ（Spine §6） | 主フォーマット |
|---|---|---|
| 獲得: 悩みの瞬間に見つかる | ループ1（コンテンツ） | SEO記事、ショート動画、Xポスト |
| 増幅: シェア・保存で複利を生む | ループ2（UGC） | カルーセル、診断エンタメ、シェア画像 |
| 維持: 毎日の接触習慣を作る | ループ4（リテンション） | 今日の運勢、LINE配信、メルマガ |

### 1.3 トーン&マナー（全コンテンツ共通・違反は公開差し止め）

- **世界観**: 「夜空の下で、いちばん話を聞いてくれる存在」（Spine §9）
- **禁止事項**: 恐怖訴求（「〜しないと不幸になる」）、断定的不幸予言、依存誘発（「毎日見ないと運気が下がる」）、医療・法律判断の代替を示唆する表現
- **推奨**: 「〜かもしれません。だから今日は〜してみては」の提案形。必ず「明日の一歩」で終える（提供価値の本質＝意思決定の後押し、Spine §1）

## 2. コンテンツピラー（5本柱）

全コンテンツは以下5ピラーのいずれかに属する。ピラー外の企画は原則作らない（リソース分散防止）。

| # | ピラー | 目的 | 対応ループ | 主KPI | 制作比率目安 |
|---|---|---|---|---|---|
| ① | 今日/今週/今月の運勢 | 習慣化・毎日の接触 | リテンション | 開封率/再訪率 | 20% |
| ② | 悩み別ガイド | SEO/AEO資産・獲得 | コンテンツ | 検索流入/登録CVR | 30% |
| ③ | 診断・占いエンタメ | バイラル・認知 | UGC | シェア数/完視聴率 | 25% |
| ④ | ユーザーストーリー/UGC | 社会的証明・信頼 | UGC/リファラル | 保存率/引用数 | 15% |
| ⑤ | 占い教養・裏側 | 信頼・ブランド・E-E-A-T | コンテンツ/ブランド | 指名検索/被リンク | 10% |

### 2.1 ピラー①: 今日/今週/今月の運勢（習慣化）

- **配信面**: X（毎朝7:00）、LINE（毎朝7:30）、アプリ内、Instagramストーリーズ、メルマガ（週次）
- **設計原則**: 単なる吉凶ではなく「今日の一歩」を必ず含める（例: 「牡羊座: 迷っていた連絡、今日の午前が追い風」）。**夜のふりかえり導線**（「今日の運勢、当たってた？」）とセットでリテンションループを閉じる
- **制作**: AI自動生成→人間の品質チェック（§6.2のフロー）。1日あたり12星座×3面（X/LINE/アプリ）を30分以内で確定する運用
- **判定文**: 朝運勢経由の当日再訪率が15%を下回る月は、文面フォーマットのA/Bを翌月2本以上実施する

### 2.2 ピラー②: 悩み別ガイド（恋愛・仕事・人間関係のSEO/AEO資産）

- **配信面**: オウンドメディア（正本）、YouTube長尺（転用）、Pinterest（アイキャッチ転用）
- **クラスタ構造・キーワード戦略**: [13_SEO_AEO_LLMO.md](./13_SEO_AEO_LLMO.md) §3を正本とする
- **テーマ例**: 「復縁 タイミング」「片思い 諦めるべきか」「転職 迷い 決められない」「職場 人間関係 疲れた」— **悩みが顕在化した検索**を捕捉し、記事末尾で「この悩みをあなたの状況で占う」CTAへ
- **品質基準**: 監修者クレジット必須（§4.1）、一次情報（自社鑑定データの匿名統計）を1記事1つ以上含めることを推奨
- **判定文**: 公開90日時点で検索流入月100セッション未満かつ登録CVR 4%未満の記事はリライト対象キューに入れる

### 2.3 ピラー③: 診断・占いエンタメ（バイラル用）

- **配信面**: TikTok、Instagramリール、YouTubeショート、X
- **設計原則**: 「参加できる」「自分の結果を言いたくなる」「友達をタグ付けしたくなる」の3条件のうち2つ以上を満たす企画のみ制作
- **企画例**: 「生年月日でわかる、あなたが恋愛で無意識にやってるクセ」「12星座、別れ際の言動ランキング」「コメントに星座を書くと今週の注意日を返信」
- **判定文**: 4週間で平均シェア率（シェア数/リーチ）が0.5%未満の企画シリーズは打ち切り、上位シリーズへ制作枠を移す

### 2.4 ピラー④: ユーザーストーリー/UGC（社会的証明）

- **配信面**: Instagram（リポスト・ストーリーズ）、オウンドメディア（体験談記事）、X（引用）
- **内容**: 「鑑定で背中を押されて行動した話」を許諾を得て編集・紹介（§5.3の許諾フロー厳守）。**誇大な的中体験ではなく「気持ちが整理された・一歩踏み出せた」体験**を選ぶ（Spine §1の提供価値と一致させる）
- **判定文**: 体験談コンテンツの保存率がピラー②平均を上回ることを維持する。下回る場合は選定基準（具体性・共感性）を見直す

### 2.5 ピラー⑤: 占い教養・裏側（信頼・ブランド）

- **配信面**: オウンドメディア、YouTube長尺、PR素材
- **内容**: タロット・占星術の歴史と仕組み、AI鑑定の仕組みの透明な解説（「AIはこう占っている」）、監修者インタビュー、「占いとの健全な付き合い方」（倫理ポジションの発信そのものが差別化 — Spine §9）
- **判定文**: 本ピラーはPV目標を持たない。評価は被リンク獲得数・メディア引用数・指名検索増分で行う（`13` §7）

## 4. フォーマット別設計

### 4.1 オウンドメディア記事

**構成テンプレート（ピラー②標準型）**

| 順序 | ブロック | 要件 |
|---|---|---|
| 1 | 結論ファースト（150字以内） | 検索意図への直接回答。AEO対応（`13` §4） |
| 2 | 共感導入 | 「あなたは今〜という状態かもしれません」読者の状況の言語化 |
| 3 | 本論（H2×3-5） | 各H2冒頭に要点1文（強調スニペット対策）。表・リスト活用 |
| 4 | 占い的視点 | タロット/占星術ならこの悩みをどう見るか（差別化パート） |
| 5 | 明日の一歩 | 具体的アクション3つ（ブランドの「背中を押す」体現） |
| 6 | CTA | 「この悩みをあなたの状況で占う」→鑑定 or LINE登録 |
| 7 | FAQ（3-5問） | FAQスキーマ実装（`13` §4） |
| 8 | 監修者・出典 | E-E-A-T対応 |

**E-E-A-T・監修体制**

- 全記事に執筆者名＋プロフィール、ピラー②⑤は**外部監修者**（占術家・臨床心理士等、契約2名以上）のクレジットを付与
- 心理・健康に触れる記述は監修必須。医療的内容は書かない（「つらい場合は専門機関へ」の定型導線を挿入）
- 運営者情報・編集ポリシー・監修一覧ページを公開（`13` §3.5と共通実装）

**品質チェックリスト（公開前に全項目Yes必須）**

- [ ] 検索意図に150字以内で直接回答しているか
- [ ] 恐怖訴求・断定的不幸予言・依存誘発表現がないか
- [ ] 「明日の一歩」が具体的行動として書かれているか
- [ ] 一次情報（自社統計・監修コメント）が1つ以上あるか（ピラー②）
- [ ] 構造化データ（Article/FAQ）が実装されているか
- [ ] CTAが記事の悩みと直結した鑑定テーマにリンクしているか

### 4.2 ショート動画（TikTok / リール / YouTubeショート）— 台本テンプレ5型

全型共通ルール: 冒頭1.5秒で「自分ごと化フック」、長さ15-35秒、字幕100%表示、最後は「プロフィールのリンクから自分のを占える」で統一。

**型1: 共感フック型**

```
[0-2秒]  フック: 「連絡が来ない夜、何回もスマホ見ちゃう人」
[2-15秒] 展開: その心理の言語化 →「それって『答えが欲しい』サインです」
[15-25秒] 転換: 占いでの見方を一言（タロットなら◯◯のカードの状態）
[25-30秒] CTA: 「あなたの『答え』、プロフィールから1分で占えます」
```

**型2: 診断参加型**

```
[0-2秒]  フック: 「生年月日の“日”が奇数の人だけ見て」
[2-20秒] 展開: 分岐別の結果を順に発表（自分の番を待たせる＝完視聴設計）
[20-30秒] CTA: 「コメントに星座を書くと今週の注意日を返信します」
```

**型3: あるある型**

```
[0-2秒]  フック: 「蠍座が本気で怒ってるときの行動」
[2-20秒] 展開: あるある3連発（3つ目に意外性を置く）
[20-30秒] CTA: 「保存して友達の星座と見比べて」→タグ付け誘発
```

**型4: ビフォーアフター型**

```
[0-2秒]  フック: 「1ヶ月前、転職を決められなかった私へ」
[2-20秒] 展開: 鑑定で気持ちが整理された過程（UGC/体験談ベース、許諾済み）
[20-30秒] CTA: 「迷ってる人は、一度言葉にしてもらうと楽になるよ」
```

**型5: ASMR鑑定型**

```
[0-2秒]  フック: カードシャッフル音+「今夜、これを見たあなたへ」
[2-25秒] 展開: 静かな環境音でカードを引き、囁き系ナレーションで一言鑑定
[25-35秒] CTA: 「今夜のあなた専用の鑑定はリンクから」
```

- **運用判定**: 各型を毎週均等に投稿→4週ごとに完視聴率・プロフィール遷移率で型の配分を再加重（最上位型に40%まで集中可）

### 4.3 カルーセル画像（Instagram / Pinterest転用）

- **標準構成（8-10枚）**: 表紙（保存したくなるタイトル）→ 問題提起 → 本論1枚1メッセージ×5-7 → まとめ1枚（スクショ保存用の総括図）→ CTA
- **デザイン要件**: 夜空トーンのブランドテンプレート使用（`09_Brand_Strategy.md`）、1枚あたり文字数60字以内、まとめ枚は単体でシェアされても文脈が通じること
- **判定文**: 保存率3%未満のテーマは表紙コピーを変えて1回だけ再投稿テスト。再度3%未満ならテーマ廃棄

### 4.4 X投稿（毎日の12星座ポストの型）

**毎朝7:00の定型（自動生成→人間確認）**

```
【7/11(土) 12星座の今日】
1位 ♌獅子座: 迷っていた誘い、乗ってみて。夜にいい報せ
2位 ♎天秤座: …
（3-10位は簡潔に）
11位 ♑山羊座: 焦らなくていい日。夜の入浴が回復スイッチ
12位 ♓魚座: 決断は明日へ。今日は「決めない」が正解
▼あなたの詳しい今日はこちら（リンク）
```

- **設計意図**: ランキング形式は「自分の順位に言及したくなる」＝引用RT誘発（`05` §2.5）。下位星座ほど**優しい救済文**にする（脅さない — 下位でもスクショしたくなる言葉が本サービスの差別化）
- **その他の型**: 週1の「今週の新月/満月」解説、週2のバズ狙い診断ポスト（`08_Viral_Strategy.md` §4のフック集）

### 4.5 LINEリッチメッセージ

- **朝運勢（毎日7:30）**: 画像1枚+一言+「詳しく見る」ボタン。タップで当日の詳細鑑定へ
- **週次セグメント配信（週2まで）**: 悩みカテゴリタグ（恋愛/仕事/人間関係）別に出し分け。A/Bは常時1本以上稼働
- **設計原則**: 配信頻度上限・ブロック率管理は [11_CRM.md](./11_CRM.md) を正本とする。コンテンツ側の責務は「開封したくなる1行目」と「タップ先の鑑定体験との一貫性」

### 4.6 メールマガジン

- **役割**: LINE未登録層・離脱復帰層への長文チャネル。週1回「今週の星模様と、あなたへの3つの問い」
- **構成**: 今週の占星術トピック → 悩み別ワンポイント → 新機能/読み物紹介 → 鑑定CTA
- **判定文**: 開封率25%未満が4週続いたら件名戦略を全面改定。配信解除率が単月1%を超えたら頻度を隔週化

## 5. UGC戦略

### 5.1 ハッシュタグ設計

| 階層 | タグ | 用途 |
|---|---|---|
| ブランドタグ | #本サービス名（命名確定後に置換） | 全UGCの集約・計測の主キー |
| 体験タグ | #(サービス名)やってみた | 診断参加系の投稿導線 |
| 結果タグ | #今日の(サービス名) | 朝運勢のスクショ投稿用 |
| キャンペーンタグ | 企画ごとに期間限定発行 | 施策別計測 |

- シェア画面にタグを**自動挿入**（コピー1タップ）。タグ非依存の言及も週次でエゴサーチ計測（`14_KPI_Dashboard.md`）
- ブランド名確定（`09_Brand_Strategy.md`）の際、**タグとして短く・誤変換されにくい**ことを命名要件に含める

### 5.2 スクショ映えする結果画面（プロダクト要件）

> 実装要件は [07_UX_Improvement.md](./07_UX_Improvement.md)、シェア導線の詳細は [08_Viral_Strategy.md](./08_Viral_Strategy.md) を正本とする。コンテンツ戦略からの要求仕様は以下。

| 要件 | 判定基準 |
|---|---|
| 結果画面はそのままスクショしてストーリーズに貼れる縦長比率（9:16セーフエリア） | デザインQAで全鑑定テーマ確認 |
| 「刺さる一文」が画面上部に大きく表示される（言葉が主役） | 1テーマにつき出力バリエーション20種以上 |
| サービス名・ロゴが控えめかつ必ず写り込む位置にある | スクショ拡散時の出所判別率100% |
| ネガティブな結果画面を作らない（下位・注意日も救済文で締める） | 倫理レビュー（Spine §9） |
| シェア用画像の自動生成（ストーリーズ用/フィード用/X用の3サイズ） | シェアボタンから2タップ以内 |

### 5.3 UGCの公式二次利用: 許諾フローとリポスト運用

**許諾フロー（ステマ規制・著作権対応）**

1. 対象投稿の発見（タグ巡回: 毎営業日）
2. 公式アカウントからコメントまたはDMで定型許諾文を送付（「素敵な投稿をありがとうございます。公式アカウント/広告でご紹介してもよいですか？『OK』の返信で承諾となります」）
3. 承諾の記録（スプレッドシート: 投稿URL・承諾日時・利用範囲）
4. 利用時のクレジット表記（@ユーザー名）必須
5. 広告転用（`05` §4.2）は**別途明示的な承諾**を取得（オーガニックリポスト承諾と区別する）

**リポスト運用**

- Instagramストーリーズ: 良質UGCを毎日1-3件リポスト（リポストされる体験自体がUGC投稿の報酬になる）
- 月次「今月のベスト投稿」をフィード紹介+チケット進呈（アンバサダー発掘の入口、`05` §2.11）
- **判定文**: リポストされたユーザーの30日以内再投稿率50%以上を維持する（リポストが報酬として機能している証拠）

### 5.4 UGC発生量の目標

| 時点 | 月間UGC投稿数（タグ計測） | SNS投稿率（投稿UU/MAU） |
|---|---|---|
| 6ヶ月 | 3,000 | 2% |
| 12ヶ月 | 15,000 | 3%（Spine §5） |
| 36ヶ月 | 40万 | 6%（Spine §5） |

### 7.2 NSM（WRS）との接続 — 「保存率」を先行指標として扱う

- WRSの定義（Spine §3）は「鑑定完了＋保存/シェア/メモ/再訪」。コンテンツにおける**保存行動はWRSと同じ心理**（=あとで見返したい・自分ごと化）の現れである
- したがってコンテンツの最上位評価軸は**保存率**とし、四半期ごとに「コンテンツ保存率と、そのコンテンツ経由ユーザーのWRS転換率」の相関を分析する
- **判定文**: 相関が確認できたフォーマットは保存率で日常管理してよい。相関が弱い（r<0.3）フォーマットは登録CVR・WRS転換の直接計測に切り替える

### 7.3 レポーティング

- 日次: 公開本数、朝運勢の開封・再訪（自動ダッシュボード）
- 週次: フォーマット別KPI、型別ランキング、UGC発生数（編集会議のインプット）
- 月次: コンテンツ経由の登録数・WRS転換・制作単価、ピラー別配分の実績と補正
- 四半期: 保存率×WRS相関分析、コンテンツ資産の棚卸し（リライト/統合/削除の判定）
