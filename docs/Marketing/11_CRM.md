## 1. CRM総論 — 「CRMは売り込みではなく、関係の記憶」

### 1.1 基本思想

本サービスのブランドの核は「夜空の下で、いちばん話を聞いてくれる存在」である（→ [09_Brand_Strategy.md](./09_Brand_Strategy.md)）。CRMはこの世界観をチャネル横断で実装する装置であり、次の3原則に従う。

1. **CRMは「関係の記憶」である**。前回の相談内容・悩みカテゴリ・記念日・ストリークを覚えていて、それを踏まえて声をかける。「覚えていてくれた」という体験そのものがリテンションドライバーであり、汎用一斉配信はこの原則に反する
2. **1配信 = 1つの価値**。すべての配信は「ユーザーが受け取って得をする情報・体験」を必ず1つ含む。売り込みだけの配信は原則禁止（例外: 領収書等の取引通知）
3. **恐怖訴求・依存誘発の禁止**。「今行動しないと運気が下がる」「見ないと不幸になる」型の文面は、開封率が上がるとしても使用しない。これは倫理原則であると同時に、解約率4%・NPS 50を実現する長期資産である（→ Spine §9）

### 1.2 CRMが担うKPI（Spine §5との対応）

| Spine KPI | 12ヶ月目標 | CRMの寄与仮説 |
|---|---|---|
| D1継続率 | 45% | ウェルカムD1配信でD1復帰を+8pt押し上げる（ホールドアウト比） |
| D7継続率 | 25% | ウェルカムシリーズ+朝夜習慣化通知でD7を+5pt押し上げる |
| D30継続率 | 15% | ストリーク維持+離脱リスク介入でD30を+4pt押し上げる |
| 無料→有料転換率 | 6% | 初課金促進シナリオが全転換の35%を創出する |
| 月次解約率 | 7%→4% | 解約防止+利用促進で解約率を-1.5pt改善する |
| WRS（NSM） | — | CRM経由セッションのWRS化率50%以上（開かせるだけでなく共鳴させる） |

### 1.3 LINEを中核チャネルとする理由

| 観点 | 判定根拠 |
|---|---|
| 日本市場の到達率 | LINEは日本の月間利用者が9,000万人超と推定され、コアターゲット（20〜34歳女性）のほぼ全員が日常利用している。アプリ未インストール層にも到達できる唯一のデイリーチャネル |
| 開封率 | LINE公式アカウントのメッセージ開封率は一般に60%前後と言われ、メール（15〜25%）の2〜4倍。デイリー接点の主戦場として最も効率が高い |
| ミニアプリ連携 | LIFF（LINE Front-end Framework）により、友だち追加→そのまま鑑定→結果保存までLINE内で完結できる。「アプリDLの壁」を迂回した鑑定体験の入口になる |
| リッチメニュー | 常設のナビゲーション（今日の運勢/相談/マイページ）が「毎朝開く場所」として習慣化ループ（Spine §6-4）の起点になる |
| 相性・紹介との親和性 | トーク画面からの共有が1タップで、リファラルループ（→ [08_Viral_Strategy.md](./08_Viral_Strategy.md)）の主要導線になる |

**チャネル戦略の結論**: LINE = デイリー接点の中核、プッシュ = アプリ利用者の習慣化トリガー、メール = 週次ダイジェストと取引通知、アプリ内メッセージ = 文脈オファー。同一メッセージの多重配信は行わず、後述の優先順位ルール（§6.3）で1ユーザー1メッセージに正規化する。

## 2. チャネル別役割定義

### 2.1 役割マトリクス

| チャネル | 主な役割 | 得意な場面 | 使わない場面 | 目標指標 |
|---|---|---|---|---|
| LINE公式 | デイリー接点・復帰導線・キャンペーン | 朝の運勢、休眠復帰、相性占い招待 | 領収書等の法的通知 | 開封率55%以上 / ブロック率 月0.8%未満 |
| プッシュ通知（アプリ） | 習慣化トリガー・リアルタイム性 | 朝夜のリマインド、ストリーク、鑑定完了通知 | 長文の説明、キャンペーン詳細 | オプトイン率60%以上 / 開封率8%以上 |
| メール | 週次ダイジェスト・重要なお知らせ・領収書 | 週報、料金改定、領収書、規約変更 | 即時性が必要な通知 | 開封率25%以上 / 配信停止率 月0.3%未満 |
| アプリ内メッセージ | 文脈オファー・機能案内 | 鑑定直後のアップセル、新機能ツアー | アプリ外のユーザーへの連絡 | 表示→タップ率15%以上 |

### 2.2 チャネル別の運用規定

**LINE公式アカウント**
- リッチメニューは6分割固定: 「今日の運勢」「相談する」「相性を占う」「マイページ」「ヘルプ」「プラン」
- 応答メッセージ（友だち追加直後）は自動で「初回鑑定への導線+通知設定の選択UI」を返す
- セグメント配信を原則とし、全体一斉配信は月2回まで（大型アップデート・季節イベントのみ）
- 通数コスト管理: 配信対象は「直近90日以内にアクティブ or 復帰シナリオ対象」に限定し、無反応180日超のユーザーへの定期配信は停止する

**プッシュ通知**
- オンボーディングD0で「朝の運勢通知の時刻」をユーザー自身に選ばせる（デフォルト7:30）。自己選択させることでオプトイン率と開封率が上がる
- 通知は必ずディープリンクを持ち、タップ後2画面以内で価値に到達させる

**メール**
- 週次ダイジェスト（毎週日曜20:00）: 今週の相談ふりかえり+来週の運勢サマリ+保存済み鑑定のハイライト
- 取引メール（領収書・プラン変更・解約確認）はマーケティング配信停止の対象外として送達を保証する

**アプリ内メッセージ**
- 表示条件は「その画面の文脈と一致する場合のみ」。例: 深掘り鑑定の閲覧完了直後にのみチケット案内を出す
- 同一ユーザーへのモーダル表示は1日1回まで。バナー/カード型は2枠まで

## 3. ライフサイクルステージ定義

### 3.1 ステージと判定条件（数値定義）

すべての判定はイベントデータから日次バッチ+リアルタイムイベントで自動更新する（§10）。1ユーザーは常に1つの主ステージに属し、課金状態（無料/有料）を副軸として持つ。

| # | ステージ | 判定条件（すべて数値・判定文） | 遷移先 |
|---|---|---|---|
| S1 | 新規（Onboarding） | 登録からの経過日数 ≤ 7日 | S2 / S6 |
| S2 | 活性化（Activated） | 登録8日以降 かつ 初回鑑定完了済み かつ 直近7日ログイン日数 ≥ 1 | S3 / S6 |
| S3 | 習慣化（Habituated） | 直近14日のログイン日数 ≥ 8（週4相当） かつ 直近14日の鑑定完了 ≥ 5 | S4 / S6 |
| S4 | 課金（Paying） | 有効なサブスク保有 または 直近30日にチケット購入 ≥ 1 | S5 / S6 / S8 |
| S5 | ロイヤル（Loyal） | サブスク継続 ≥ 3ヶ月 かつ 直近30日WRS ≥ 8 かつ（紹介実績 ≥ 1 または 保存/シェア率 ≥ 30%） | S6 / S8 |
| S6 | 離脱リスク（At-Risk） | 直近7日ログインなし かつ 過去28日中に週3日以上利用した週が ≥ 2 | S7 / S2〜S5復帰 |
| S7 | 休眠（Dormant） | 直近30日ログインなし | S8'（復帰） |
| S8 | 解約（Churned-Paid） | サブスク解約済み かつ アプリ/LINE利用は継続の可能性あり | S4（win-back） |
| S8' | 復帰（Resurrected） | 休眠（S7）から再ログインしてからの経過日数 ≤ 14日 | S2 / S6 |

### 3.2 遷移ルールの補足

- **昇格は即時、降格は日次バッチ**で判定する（例: 課金完了イベントで即S4へ、S6への降格は毎朝5:00のバッチ）
- S6（離脱リスク）の定義は「過去に習慣があった人が止まった」ことを検出する設計。単に低頻度なユーザー（週1利用など）はS2のまま扱い、離脱リスク介入の対象にしない — 週1が本人のペースである場合に介入すると疲弊を招くため
- S8（解約）ユーザーがS3相当の利用を続けている場合、「利用は続けたいが価格が合わない」シグナルとしてwin-backシナリオでLightプラン（¥980）を優先提示する（→ [10_Pricing.md](./10_Pricing.md)）
- 各ステージの人数・遷移率は [14_KPI_Dashboard.md](./14_KPI_Dashboard.md) のライフサイクルファネルで週次モニタリングする

### 3.3 ステージ別のCRMゴール

| ステージ | CRMゴール（判定文） | 主要シナリオ |
|---|---|---|
| S1 新規 | D7までに「鑑定3回完了 かつ 保存1回」に到達させる | ウェルカムシリーズ（§4.1） |
| S2 活性化 | 14日以内に週4利用（S3昇格）に到達させる | 習慣化・朝夜通知（§4.3） |
| S3 習慣化 | 30日以内に初課金（S4昇格）率12%を達成する | 初課金促進（§4.2） |
| S4 課金 | 月次解約率4%以下を維持する | 利用促進+解約防止（§4.6） |
| S5 ロイヤル | 紹介実績を1人あたり年2件以上にする | 記念日+紹介（§4.8） |
| S6 離脱リスク | 7日以内の復帰率40%を達成する | 離脱リスク介入（§4.4） |
| S7 休眠 | 90日以内の復帰率15%を達成する | 休眠復帰（§4.5） |
| S8 解約 | 90日以内の再課金率12%を達成する | win-back（§4.7） |

## 4. ステージ別シナリオ設計

各シナリオは「トリガー条件 / チャネル / タイミング / メッセージ例文 / 目標KPI」を必ず定義する。文面はすべて [09_Brand_Strategy.md](./09_Brand_Strategy.md) のトーン（あたたかい・具体的・脅さない・押しつけない）に準拠し、§8の禁止表現チェックを通過したもののみ配信する。

### 4.1 ウェルカムシリーズ（S1: D0/D1/D3/D7）

**目的**: D1 45%・D7 25%の達成。初回7日間で「話を聞いてもらえる場所」という体験記憶を作る。

| 配信 | トリガー | チャネル | タイミング | 目標KPI |
|---|---|---|---|---|
| W-0 | 登録完了 かつ 初回鑑定完了 | LINE（未友だちならアプリ内+メール） | 完了から30分後 | 開封率65% / 保存アクション率20% |
| W-0b | 登録完了 かつ 初回鑑定**未**完了 | LINE/プッシュ | 登録から2時間後 | 初回鑑定完了率+15pt |
| W-1 | 登録翌日 | プッシュ+LINE（プッシュ未達時のみLINE） | ユーザー選択の朝時刻（デフォルト7:30） | D1復帰率45% |
| W-3 | 登録3日目 かつ 鑑定2回以上完了 | LINE | 20:00〜21:00 | 相性占い着手率10% |
| W-3b | 登録3日目 かつ 鑑定1回以下 | LINE | 20:00〜21:00 | 再鑑定率25% |
| W-7 | 登録7日目 | LINE+メール（週次ダイジェスト初回を兼ねる） | 日曜なら20:00、平日なら19:00 | D7復帰率25% / 週報開封率30% |

**メッセージ例文**

- W-0（初回鑑定完了後）:
  > 今日は話してくれてありがとうございました。「{悩みカテゴリ}」のこと、ちゃんと覚えています。今日の鑑定は保存しておけるので、迷ったときにいつでも読み返してくださいね。
- W-0b（未完了）:
  > はじめまして。まだ少し緊張しますよね。最初は「今日の運勢」をみるだけでも大丈夫です。1分で終わるので、よかったらどうぞ。
- W-1（翌朝）:
  > おはようございます。今日の{星座}のあなたは「{今日のひとことサマリ}」。昨日話してくれた{悩みカテゴリ}にも少し関係がありそうです。
- W-3（順調）:
  > 3日間おつかれさまです。ところで、気になる人や友だちとの「相性」も占えるのをご存じですか？ 2人分の星の重なりから、関係のヒントが見えます。
- W-7:
  > 1週間、おつかれさまでした。今週あなたが話してくれたことを、1枚のふりかえりにまとめました。来週のあなたの星模様も添えています。

**設計上の判定文**: W-1の配信時刻はユーザーがD0に自己選択した時刻を最優先する。未選択の場合のみデフォルト7:30。W-3はA/Bで「相性訴求」vs「深掘り鑑定訴求」を検証する（→ [15_Experiment_Backlog.md](./15_Experiment_Backlog.md)）。

### 4.2 初課金促進（S2/S3 → S4）

**目的**: 無料→有料転換率6%（12ヶ月）→10%（36ヶ月）。CRM経由で全転換の35%を創出する。

**原則**: 課金オファーは「価値体験の直後」にのみ出す。利用実績のないユーザーへの割引プッシュは行わない（依存誘発・押しつけの回避、かつ転換の質が低いため）。

| 配信 | トリガー | チャネル | タイミング | オファー | 目標KPI |
|---|---|---|---|---|---|
| P-1 | 無料鑑定の月間上限に初到達 | アプリ内メッセージ | 上限到達の画面遷移時 | Light ¥980の案内（割引なし） | 表示→購入率6% |
| P-2 | 深掘り鑑定のプレビューを2回以上閲覧 | アプリ内+LINE（24時間後） | 閲覧直後/翌日20:00 | チケット1枚 or Standard ¥2,980 | CVR 8% |
| P-3 | S3昇格（習慣化）から7日経過 かつ 未課金 | LINE | 昇格7日後の20:30 | Standard 7日間トライアル | トライアル開始率10% / 有料転換率50% |
| P-4 | トライアル終了48時間前 | プッシュ+メール | 終了48時間前の12:00 | 継続確認（自動課金の明示） | 継続率60% / 苦情ゼロ |

**メッセージ例文**

- P-2（LINE、翌日）:
  > 昨日みていた「{深掘りテーマ}」の鑑定、続きが気になっていたら。あなたの出生図をもとに、{悩みカテゴリ}を30分かけてじっくり読み解きます。必要なときに使えるチケットもあります。
- P-3:
  > この2週間、ほとんど毎日会えてうれしいです。もしよければ、7日間だけStandardのすべての機能を試してみませんか。合わなければ、そのまま無料のままで大丈夫です。
- P-4（トライアル終了前）:
  > Standardのお試しが、あさって{日付}に終わります。このまま続ける場合は月額¥2,980です。続けない場合は操作は不要で、自動で無料プランに戻ります。

**判定文**: P-4のような「課金前の明示的な事前通知」は必須とする。ダークパターン（気づかぬ自動課金）は解約率とブランド毀損のコストが転換益を上回ると判断する。

### 4.3 ストリーク維持（S2/S3）

**目的**: リテンションループ（朝の運勢→夜のふりかえり→ストリーク→翌朝再訪）の駆動。D30 15%→25%への寄与。

| 配信 | トリガー | チャネル | タイミング | 目標KPI |
|---|---|---|---|---|
| ST-1 朝の運勢 | 通知オプトイン済み | プッシュ（アプリ）/LINE（非アプリ） | ユーザー選択時刻（デフォルト7:30） | 開封率12% / 開封→鑑定完了率60% |
| ST-2 夜のふりかえり | 当日朝の運勢を閲覧済み かつ 21:30時点で夜の記録なし | プッシュ | 21:30 | 記録完了率25% |
| ST-3 ストリーク危機 | ストリーク7日以上 かつ 当日22:00時点で未ログイン | プッシュ | 22:00 | 当日中復帰率30% |
| ST-4 マイルストーン | ストリーク7/30/100日到達 | アプリ内+LINE | 到達時 | シェア率8%（→ 08_Viral_Strategy.md） |

**メッセージ例文**

- ST-1: 「おはようございます。今日の{星座}は『{ひとこと}』。3行だけ、読んでいってください。」
- ST-3: 「今日も一日おつかれさまでした。{N}日続いているあなたの記録、今日の分がまだ空いています。ひとことだけでも残していきませんか。」
- ST-4: 「{N}日間、毎日ここに来てくれました。あなたの{N}日分の星の記録を、1枚の星図にまとめました。」

**禁止事項（判定文)**: ストリーク喪失を脅しに使わない。「途切れたら台無し」「連続記録が消えます！」型の文面は禁止。ST-3は「記録が空いている」という事実の提示に留める。ストリークが途切れたユーザーには「おかえりなさい。記録はここから続きです」と再開を軽くする文面を返す（喪失感の演出をしない）。

### 4.4 離脱リスク介入（S6）

**目的**: 「直近7日ログインなし かつ 過去28日中に週3日以上利用した週が2週以上」のユーザーを7日以内復帰率40%で取り戻す。

| 配信 | トリガー | チャネル | タイミング | 内容 | 目標KPI |
|---|---|---|---|---|---|
| AR-1 | S6判定当日 | プッシュ | 判定当日の、本人の過去最頻利用時間帯 | 未読の運勢+前回相談の続き | 復帰率18% |
| AR-2 | S6判定から3日、未復帰 | LINE | 20:00 | 「あなた宛の1枚鑑定」（AI生成のパーソナル鑑定カード） | 復帰率12% |
| AR-3 | S6判定から6日、未復帰 | メール | 19:00 | 不在期間の星の動きダイジェスト+通知設定見直し導線 | 復帰率6% / 配信設定変更率3% |

**メッセージ例文**

- AR-1: 「少し間があきましたね。{前回の悩みカテゴリ}のこと、その後どうなったかなと思っていました。今日の星まわりを1枚にまとめてあります。」
- AR-2: 「あなたの出生図をもとに、いまの時期だけの1枚鑑定を作りました。読むのに1分もかかりません。よかったら。」
- AR-3（メール件名）: 「この1週間の、あなたの星の動きをまとめました」

**判定文**: AR-3で復帰しなかった場合、追加の介入は行わずS7（休眠）バッチに引き継ぐ。介入は最大3通で打ち切る — 4通目以降の限界復帰率は1%未満と想定され、ブロック率上昇のコストが上回る。

### 4.5 休眠復帰（S7: 7日/30日/90日）

**目的**: 休眠期間の長さに応じてメッセージの前提を変える。長期休眠ほど「再入門」として扱う。

| 配信 | トリガー | チャネル | タイミング | 訴求内容 | 目標KPI |
|---|---|---|---|---|---|
| DR-7 | 最終ログインから7日（S6シナリオ完了者は除外しS6→S7連続介入を防ぐ） | LINE | 20:00 | 前回文脈の継ぎ直し | 復帰率10% |
| DR-30 | 最終ログインから30日 | LINE+メール | 土曜11:00 | 新機能・変わったことの紹介+今週の運勢 | 復帰率5% |
| DR-90 | 最終ログインから90日 | メール（LINEブロック済み想定でメール主軸） | 月初の土曜11:00 | 「はじめまして」に近い再入門+無料鑑定1回分 | 復帰率2.5% / 復帰後D7残存率30% |

**メッセージ例文**

- DR-7: 「1週間ぶりです。あのあと{悩みカテゴリ}はどうなりましたか。話の続きは、いつでもここから再開できます。」
- DR-30: 「1ヶ月のあいだに、こちらでは少し変わったことがありました。{新機能1行}。今週のあなたの運勢と一緒に、のぞいてみてください。」
- DR-90（メール件名）: 「おひさしぶりです。また夜空の下で話しませんか」
  > 本文: 3ヶ月ぶりのご案内です。あれから機能もずいぶん増えました。もしまた話したくなったときのために、無料の鑑定を1回分ご用意しています。もう案内が不要な場合は、下のリンクからいつでも停止できます。

**判定文**: DR-90を最後に定期的な復帰配信は停止する（年2回の大型アップデート告知のみ例外）。「復帰後D7残存率30%」を復帰の質の指標とし、開封率だけでの成功判定を禁止する。

### 4.6 解約防止（S4/S5の解約意向検知）

**目的**: 月次解約率7%→4%。解約は「引き止め」でなく「原因の解消」で防ぐ。

| 配信 | トリガー | チャネル | タイミング | 内容 | 目標KPI |
|---|---|---|---|---|---|
| CS-1 利用低下検知 | 課金者 かつ 直近14日の鑑定完了 ≤ 2（前月同期間比50%以下） | LINE | 検知翌日20:00 | 使い方の再提案（残チケット・未体験機能） | 利用回復率25% |
| CS-2 更新前価値通知 | 更新日7日前 | メール | 12:00 | 当月の利用サマリ（相談N回・保存N件・節目） | 解約意向率の低減（対照比-1pt） |
| CS-3 解約フロー内 | 解約ページ到達 | アプリ内 | 即時 | 理由別の代替提案（§下記） | 解約撤回率20% |
| CS-4 解約完了直後 | 解約確定 | メール | 即時 | 感謝+データ保持の説明+再開手順 | 苦情ゼロ / 30日内再開率5% |

**CS-3 理由別代替提案（判定文で分岐）**

| 解約理由（選択式） | 提案 | 禁止事項 |
|---|---|---|
| 価格が高い | Light ¥980へのダウングレード（機能差を明示） | 引き止め用の場当たり割引の乱発 |
| 使う時間がない | 週1ダイジェストモード（通知を週1に集約） | 「もったいない」等の損失煽り |
| 期待と違った | 悩みカテゴリの再設定+別の鑑定スタイル提案 | 原因を認めない定型謝罪 |
| 一時的に不要 | 最大3ヶ月の休会（データ・記録は保持） | 休会導線を見つけにくくすること |

**メッセージ例文**

- CS-2（メール件名）: 「今月、あなたが話してくれたことのまとめ」
- CS-4: 「これまで{N}ヶ月、話を聞かせてくれてありがとうございました。あなたの記録と保存した鑑定はすべてそのまま残ります。また話したくなったら、いつでも同じ場所にいます。」

**判定文**: 解約ボタンは3タップ以内で到達可能に保つ（→ [07_UX_Improvement.md](./07_UX_Improvement.md)）。解約しやすさは短期解約率を上げても、再開率とNPSで回収できるというスタンスを取る。

### 4.7 解約後のwin-back（S8）

**目的**: 解約から90日以内の再課金率12%。

| 配信 | トリガー | チャネル | タイミング | オファー | 目標KPI |
|---|---|---|---|---|---|
| WB-1 | 解約後も無料利用継続（週1以上） かつ 解約から14日 | アプリ内 | 鑑定完了直後 | Light ¥980の提示（解約理由が価格の場合） | 再課金率8% |
| WB-2 | 解約から30日 かつ 深掘りプレビュー閲覧あり | LINE | 閲覧翌日20:00 | チケット単品（サブスク以外の選択肢） | 購入率5% |
| WB-3 | 解約から60日 | メール | 土曜11:00 | 復帰者向け30日間Light割引（¥490、初回解約者のみ1回限り） | 再課金率4% |

**メッセージ例文**

- WB-1: 「いまも来てくれてうれしいです。前に『料金が合わなくて』と教えてくれたので、月¥980のLightプランのことだけお伝えしておきますね。無理にとは言いません。」
- WB-3（メール件名）: 「また続きを話せるように、小さなプランを用意しました」

**判定文**: win-back割引は「初回解約者に1回限り」。割引待ち行動（解約→割引→再課金の反復）を防ぐため、2回目以降の解約者には割引を提示しない。

### 4.8 記念日シナリオ（S2〜S5）

**目的**: 「関係の記憶」の象徴的な体験。ロイヤル化と紹介・シェアの創出（→ [08_Viral_Strategy.md](./08_Viral_Strategy.md)）。

| 配信 | トリガー | チャネル | タイミング | 内容 | 目標KPI |
|---|---|---|---|---|---|
| AN-1 誕生日鑑定 | 登録プロフィールの誕生日当日 | LINE+アプリ内 | 当日朝（本人の通知時刻） | ソーラーリターン（1年の星図）無料鑑定 | 開封率70% / シェア率10% |
| AN-2 登録1周年 | 登録日から365日 | LINE | 当日20:00 | 1年間の相談・成長のふりかえりレポート | 開封率60% / 保存率30% |
| AN-3 初相談の記念日 | 初回鑑定完了日から365日 | アプリ内 | 当日 | 「1年前のあなたが話していたこと」の再訪カード | 閲覧率40% / WRS化率50% |

**メッセージ例文**

**判定文**: 誕生日データが未登録のユーザーには配信しない（推定生年月日での代替配信は誤爆リスクが高く禁止）。AN-1/AN-2の出力は保存・シェア可能なビジュアルカードとして生成する（UGCループへの接続）。

## 5. シナリオ全体の優先順位と競合解決

同一ユーザーが同日に複数シナリオの対象になる場合、以下の優先順位で1通に絞る（取引通知は除外）。

| 優先度 | シナリオ群 | 理由 |
|---|---|---|
| 1 | 取引・契約通知（P-4、CS-4、領収書） | 法令・信頼の根幹。抑制対象外 |
| 2 | 記念日（AN-*） | 関係の記憶の象徴。年1回しか出せない |
| 3 | 解約防止（CS-*） | 解約率への直接影響が最大 |
| 4 | 離脱リスク（AR-*）/休眠復帰（DR-*） | 復帰の時間感度が高い |
| 5 | ウェルカム（W-*） | 期間限定だが日程の再調整が可能 |
| 6 | 初課金促進（P-1〜P-3）/win-back（WB-*） | 売上系は関係系に劣後する |
| 7 | ストリーク・習慣化（ST-*） | 毎日機会があるため劣後しても損失が小さい |

## 6. 通知・配信の頻度ガバナンス

### 6.1 チャネル別上限（ハードリミット）

| チャネル | 上限 | 備考 |
|---|---|---|
| プッシュ通知 | 1日2通まで / 週10通まで | ST-1（朝）+ 夜系1通が標準形。取引通知は上限外 |
| LINE | 1日1通まで / 週4通まで | リッチメニュー・応答メッセージは上限外 |
| メール | 週2通まで（うちマーケ目的は1通） | 週次ダイジェスト+取引通知が標準形 |
| アプリ内モーダル | 1日1回まで | バナー/カードは常時2枠まで |
| 全チャネル合算 | 1日3タッチまで | 優先順位表（§5）で正規化 |

### 6.2 疲弊検知と自動抑制（判定文）

| 検知条件 | 自動アクション |
|---|---|
| 個人の直近10通のプッシュ開封が0 | プッシュを週3通以下に自動減衰。ST-1のみ継続 |
| 個人の直近10通のLINE開封率が20%未満（それ以前の半分以下） | LINEマーケ配信を週1に減衰し、次回配信で頻度設定UIを提示 |
| 通知オフ操作 or LINEブロック | 該当チャネルの全マーケ配信を即時停止。他チャネルで代替補完はしない（追いかけ禁止） |
| セグメント単位で開封率が前4週平均比-30% | 該当シナリオを自動一時停止し、Growth週次会でレビュー |

### 6.3 オプトアウト設計（判定文）

- 通知設定画面には「全停止」だけでなく「朝の運勢のみ」「週1ダイジェストのみ」等の**段階的な選択肢を必ず提供**する。二択（全部 or ゼロ）は禁止
- メールフッターの配信停止はワンクリックで完了させる（ログイン要求禁止）
- 配信停止操作の完了画面で引き止めモーダルを出さない
- 頻度設定の変更はすべてのチャネルで3タップ以内で到達可能とする

## 7. パーソナライズ戦略

### 7.1 パーソナライズ軸

| 軸 | セグメント | 配信への反映 |
|---|---|---|
| 悩みカテゴリ | 恋愛 / 仕事・キャリア / 人間関係 / 自己理解 / 家族・結婚 / 金運 | 文面の話題・深掘り鑑定の提案テーマ。直近の主カテゴリ（過去30日の相談回数最多）を使用 |
| 星座・命式 | 12星座 + 出生図要素 | 朝の運勢の内容、記念日鑑定、件名の星座差し込み |
| 利用時間帯 | 朝型（5-10時）/ 昼型（11-16時）/ 夜型（17-24時）— 過去28日の最頻セッション開始帯 | 全シナリオの配信時刻を本人の最頻帯に合わせる（固定時刻指定のあるものを除く） |
| ライフサイクル | §3のS1〜S8' | シナリオそのものの出し分け |
| 課金状態・プラン | 無料 / Light / Standard / Premium | オファー内容（→ 10_Pricing.md）。Premiumにはアップセル配信をしない |

### 7.2 AIによる1to1文面生成の方針（判定文）

- **適用範囲**: パーソナル鑑定カード（AR-2）、記念日レポート（AN-*）、週次ダイジェストの要約部分。定型シナリオの骨格（トリガー・頻度・オファー）はAI生成の対象外とし、人間が設計する
- **ガードレール**: 生成文面は配信前に自動チェックを通過必須 — (1) 禁止表現辞書（§8.2）との照合、(2) 断定的な不幸予言の検知（分類器スコア閾値超えで棄却）、(3) 医療・法律・投資助言の検知で棄却。棄却時はテンプレート文面にフォールバックする
- **品質検証**: AI生成文面はテンプレート文面との恒常A/B（各10%）で開封率・WRS化率・オプトアウト率を比較し、オプトアウト率が有意に悪化した場合は即時ロールバックする
- **プライバシー**: 相談内容の要約を文面に含める場合、本人だけが見る画面（プッシュのロック画面プレビューには出さない）。プッシュの表示文には悩みの具体的内容を含めない — 「{前回の相談}のこと」の解決は本文/遷移先でのみ行う

## 8. メッセージ文体規定

### 8.1 トーン原則（09_Brand_Strategy.md 準拠）

| 原則 | Do | Don't |
|---|---|---|
| あたたかい | 「おつかれさまです」「うれしいです」等の人格ある声 | 事務的な「〜のお知らせ」の乱発 |
| 具体的 | 「3行だけ」「1分で終わる」「{N}日続いた」 | 「素敵な」「お得な」「特別な」等の曖昧形容 |
| 脅さない | 事実の提示（「今日の分がまだ空いています」） | 「運気が下がる」「手遅れになる前に」「見ないと損」 |
| 押しつけない | 「よかったら」「無理にとは言いません」+ 常に選択肢 | 「今すぐ」「絶対」「必ず」+ 逃げ道のない二択 |

### 8.2 禁止表現（配信前の自動チェック対象辞書・抜粋）

「今だけ」「急いで」「見ないと後悔」「運気が下がる/逃げる」「最後のチャンス」「あなただけ特別に（乱発）」「〜しないと不幸に」「絶対当たる」「99%の的中」— これらを含む文面はMAツール上で配信ブロックする。

### 8.3 件名・1行目の型集

| 型 | 用途 | 例 |
|---|---|---|
| 継ぎ直し型 | 復帰系（AR/DR） | 「{悩みカテゴリ}のこと、その後どうなりましたか」 |
| 贈りもの型 | 記念日・復帰インセンティブ | 「あなたの新しい1年の星図を読みました」 |
| 記録型 | 週報・月次サマリ | 「今週あなたが話してくれたことのまとめ」 |
| 小ささ提示型 | 習慣化・再開ハードル低減 | 「3行だけ、読んでいってください」 |
| 事実+選択肢型 | 課金・契約系 | 「お試しがあさって終わります。続けない場合、操作は不要です」 |

- 件名は全角25文字以内、プッシュ本文は全角40文字以内を上限とする
- 絵文字は1メッセージ2個まで。「!」の連続使用（!!）は禁止

## 9. 計測設計

### 9.1 配信別の必須計測指標

すべてのシナリオ配信は以下を配信ID単位で計測し、[14_KPI_Dashboard.md](./14_KPI_Dashboard.md) のCRMボードに自動集計する。

| 指標 | 定義 | 最低水準（下回ったら停止レビュー） |
|---|---|---|
| 到達率 | 送信成功 ÷ 送信試行 | 95% |
| 開封率 | 開封 ÷ 到達（チャネル別に定義） | LINE 40% / メール 18% / プッシュ 5% |
| CTR | タップ ÷ 到達 | LINE 8% / メール 3% / プッシュ 3% |
| 復帰率 | 配信後24時間以内のセッション開始 ÷ 到達（復帰系のみ） | シナリオ別（§4の目標値の50%） |
| WRS化率 | 配信起点セッションのうちWRS成立 ÷ 配信起点セッション | 40% |
| オプトアウト率 | 配信後72時間以内の通知オフ/ブロック/停止 ÷ 到達 | 0.5%未満（超過で自動停止） |
| 解約率影響 | 受信群vs対照群の30日解約率差（課金者向け配信のみ） | 悪化が有意なら即停止 |

### 9.2 CRM経由売上と事業目標

- **CRM経由売上の定義**: 購入前24時間以内の最終タッチがCRM配信であるトランザクション
- **目標比率**: 月商に占めるCRM経由売上 **20〜30%** を健全域とする。**35%超は「CRM依存（オーガニック動線の弱さ）」のシグナル**として、プロダクト内導線の改善課題に転換する（Spine §7-1「広告で解決する前にプロダクト改善」のCRM版）
- 無料→有料転換のCRM寄与35%、win-back再課金率12%、復帰後D7残存率30%を四半期レビューの固定アジェンダとする

### 9.3 ホールドアウト検証の運用（判定文）

- **グローバルホールドアウト**: 全ユーザーの5%を四半期単位で「マーケCRM配信を一切受けない群」に固定する（取引通知は受ける）。CRM全体の増分効果（リテンション・売上）を四半期ごとに算出し、増分が計測できないシナリオは廃止する
- **シナリオ別ホールドアウト**: 各シナリオで対象者の10%を無配信対照群とし、復帰率・転換率・解約率を差分で評価する。「開封率が高い」は成功の証拠として認めない — 対照群との行動差分のみを成功判定に用いる
- **判定基準**: 90日間で対照群との差が検出できない（有意水準5%）シナリオは停止または改修する。改修は [15_Experiment_Backlog.md](./15_Experiment_Backlog.md) にICE評価付きで起票する

## 10. MAツール要件とデータ基盤

### 10.1 MAツール選定要件（必須要件のみ）

| 要件 | 判定基準 |
|---|---|
| LINE公式アカウント連携 | Messaging API経由のセグメント配信・リッチメニュー出し分け・LIFF連携が可能 |
| マルチチャネル統合 | プッシュ/メール/LINE/アプリ内を単一ジャーニーで制御し、§5の優先順位・§6の頻度上限をルールとして実装できる |
| リアルタイムトリガー | イベント発生から5分以内にトリガー配信を実行できる |

| ホールドアウト | 配信対象からの無作為除外群の恒常設定と効果測定レポートが可能 |
| API文面差し込み | AI生成文面をAPI経由で受け取り、配信前チェック（§7.2）をフックできる |

候補としてはBraze / Insider / KARTE等のエンタープライズMA + LINE特化ツール（Lステップ等はMVP期のみ）を比較検討し、12ヶ月目標のMAU規模で再選定する。MVP期は「LINE配信+自前イベント基盤」の最小構成で§4の主要シナリオ（W/ST/AR）から実装する。

### 10.2 イベント設計（トラッキング必須イベント）

| イベント名 | 主なプロパティ | 用途 |
|---|---|---|
| `session_start` | channel(web/line/app), 時刻 | ステージ判定・時間帯セグメント |
| `reading_completed` | 占術タイプ, 悩みカテゴリ, 所要秒 | WRS判定・悩みセグメント |
| `resonance_action` | 種別(save/share/memo/revisit) | WRS成立判定（NSMの構成要素） |
| `subscription_started` / `subscription_cancelled` | プラン, 解約理由 | S4/S8遷移・CS/WBトリガー |
| `ticket_purchased` | 券種, 文脈(直前画面) | 従量課金分析・P-2最適化 |
| `paywall_viewed` | 場所, プラン表示内容 | P-1/P-2トリガー |
| `streak_updated` | 連続日数 | ST-3/ST-4トリガー |
| `notification_opened` / `message_opened` | 配信ID, チャネル | §9.1の全指標 |
| `notification_opt_changed` | チャネル, on/off, 粒度 | 疲弊検知・§6.2 |
| `referral_sent` / `compatibility_invited` | 招待経路 | S5判定・08_Viral連携 |

### 10.3 セグメント自動更新とデータフロー

```
イベント収集（app/web/LINE） → イベント基盤（CDP）
  → リアルタイム: 課金/解約/上限到達 等のトリガーを5分以内にMAへ
  → 日次バッチ（毎朝5:00）: ステージ再判定（S1〜S8'）・時間帯/悩みセグメント更新
  → MAツール: シナリオ実行（§5優先順位・§6頻度上限を適用）
  → 配信結果イベントをCDPへ還流 → 14_KPI_Dashboard.md のCRMボードへ日次集計
```

- ステージ・セグメントはCDP側を正本とし、MAツールへは同期コピーのみ置く（ツール移行リスクの遮断）
- 誕生日・出生情報は要配慮に準じる個人情報として扱い、配信文面ログには生データを残さない（セグメントIDのみ記録）

## 11. キャンペーン・季節イベント運用

ライフサイクルシナリオ（常時稼働）とは別レイヤーで、季節キャンペーンを年間カレンダーとして運用する。キャンペーン配信も§5の優先順位・§6の頻度上限の内数であり、上限の追加枠は設けない。

### 11.1 年間キャンペーンカレンダー（骨子）

| 時期 | イベント | 内容 | 主チャネル | 目標 |
|---|---|---|---|---|
| 1月 | 新年の運勢 | 年間星図の特別鑑定（無料版+Premium深掘り） | LINE全体配信（月2回枠を使用） | 休眠復帰率を平月比+50% |
| 2月 | バレンタイン相性 | 相性占いの強化週間（→ 08_Viral_Strategy.md） | LINE+アプリ内 | 相性占い実施数を平月比2倍 |
| 3-4月 | 新生活・転機 | 仕事・環境変化テーマの深掘り訴求 | セグメント配信（悩み=仕事） | チケット販売+30% |
| 6-7月 | 夏至・七夕 | 願いごと×星の特別コンテンツ | LINE+SNS連動 | シェア率10% |
| 11-12月 | 年末ふりかえり | 「今年のあなたの記録」年間レポート | LINE+メール | 保存率35% / シェア率12% |

### 11.2 キャンペーン運用の判定文

- キャンペーンは四半期に最大2本まで。常時何かのセールが走っている状態を作らない（緊急性の希釈と煽り体質化を防ぐ）
- 値引きを伴うキャンペーンは年2回以内、値引き率は30%を上限とする（→ [10_Pricing.md](./10_Pricing.md) の価格整合性ルールに従う）
- すべてのキャンペーンにシナリオ別ホールドアウト（§9.3）を適用し、増分売上がゼロなら翌年は実施しない

## 12. 法令・プラットフォーム規約の遵守

| 領域 | 遵守事項（判定文） |
|---|---|
| 特定電子メール法 | メール配信は事前同意（登録時のチェックボックス、デフォルトオフ）を取得したユーザーのみ。送信者表示・配信停止導線を全メールに記載する |

| LINEガイドライン | 医療・投資助言に該当し得る断定表現を配信しない。ブロック率が月1%を超えた場合は配信量を見直す |
| プッシュ通知（OS規約） | 通知許諾はOSダイアログの前に「プレパーミッション画面」で価値説明を行い、拒否時は30日間再要求しない |
| 景品表示法 | 割引表示は二重価格の根拠を保持する。「通常¥2,980→」の表示は直近の実売実績があるときのみ |
