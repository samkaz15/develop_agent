## 1. グロース戦略総論

### 1.1 基本命題

> **「広告で買う成長」ではなく「プロダクトとコンテンツが自ら連れてくる成長」を設計する。**

- 有料広告経由の新規獲得は **常に全体の20%以下**（Spine §2）。広告は「成長エンジン」ではなく「実験装置」と定義する
- 全チャネル施策は最終的に NSM = **WRS（Weekly Resonant Sessions）** の増加に寄与するかで評価する。流入を増やしてもWRSに変換されないチャネルは縮小する
- 判定文: **「このチャネルの流入1,000人あたりWRS転換数が全チャネル中央値の50%未満が2ヶ月続いたら、投下リソースを半減する」**

### 1.2 4つの自己強化ループの詳細設計（Spine §6の実装）

#### ループ1: コンテンツループ（SEO/AEO/LLMO）

```
悩みキーワードの構造化コンテンツ公開
  → Google検索 / AI Overviews / ChatGPT等から流入
  → 記事内の「この悩みを今すぐ占う」CTAで鑑定体験
  → 鑑定ログ・検索クエリデータから「次に書くべき悩みテーマ」を抽出
  → コンテンツ企画に還元（ループ先頭へ）
```

| 構成要素 | 定義 | 目標値（12ヶ月） |
|---|---|---|
| 入力 | 公開記事数・構造化データ実装率 | 記事600本、構造化データ実装率100% |
| 変換 | 記事→鑑定開始CVR | 8%以上 |
| 増幅 | 鑑定ログ由来の新規記事比率 | 30%以上 |
| 計測 | オーガニック経由WRS | 週次で `14_KPI_Dashboard.md` にて追跡 |

- 詳細設計は [13_SEO_AEO_LLMO.md](./13_SEO_AEO_LLMO.md) に全面委譲。本書ではチャネルポートフォリオ上の位置づけのみ扱う

#### ループ2: UGCループ（シェア設計）

```
鑑定結果の「シェアしたくなる出力」（スクショ映え・言語化の妙）
  → ユーザーがInstagramストーリーズ / X / TikTokに投稿
  → 閲覧者が「自分もやってみたい」と流入
  → 新規ユーザーが鑑定 → 新たなUGC（ループ先頭へ）
```

| 構成要素 | 定義 | 目標値（12ヶ月） |
|---|---|---|
| 入力 | SNS投稿率（投稿UU/MAU） | 3%（Spine §5） |
| 変換 | UGC 1投稿あたり平均新規流入 | 2.0人以上 |
| 増幅 | UGC経由新規のさらなる投稿率 | 一般新規の1.5倍（同質性効果） |
| 計測 | 固有ハッシュタグ投稿数（週次） | `12_Content_Strategy.md` §5参照 |

#### ループ3: リファラルループ（関係性占い）

```
相性占い・友達占いは構造上「相手」が必要
  → 「あの人との相性を見るには相手の招待 or 情報入力」
  → 招待リンク送付 → 招待された側が結果を見る
  → 招待された側も別の相手との相性を占いたくなる（ループ先頭へ）
```

| 構成要素 | 定義 | 目標値（12ヶ月） |
|---|---|---|
| 入力 | 相性占い実施率（実施UU/WAU） | 20% |
| 変換 | 招待送付→登録CVR | 35% |
| 増幅 | K-factor | 0.25（36ヶ月で0.45、Spine §5） |
| 計測 | 紹介経由新規比率 | 25%（12ヶ月）→45%（36ヶ月） |

#### ループ4: リテンションループ（習慣化）

```
毎朝の「今日の運勢」プッシュ/LINE
  → 朝の閲覧 → 夜の「ふりかえり」（今日は当たった？）
  → ストリーク（連続日数）達成 → 翌朝の再訪動機
  → D30継続率上昇 → LTV増 → プロダクト・コンテンツ再投資（ループ先頭へ）
```

| 構成要素 | 定義 | 目標値（12ヶ月） |
|---|---|---|
| 入力 | 朝運勢の開封率（LINE/プッシュ） | 40% |
| 変換 | D7継続率 | 25%（Spine §5） |
| 増幅 | D30継続率 | 15%（Spine §5） |
| 計測 | WRS/WAU比率 | 35%以上 |

### 1.3 ループ間の相互作用（システムマップ）

4つのループは独立ではなく、以下の増幅経路で結合している。

| 経路 | メカニズム | 設計上の含意 |
|---|---|---|
| リテンション→UGC | 習慣化ユーザーほど投稿率が高い（愛着×接触頻度） | シェア導線はD7以降ユーザーに重点表示 |
| UGC→コンテンツ | UGC投稿文はロングテールKW・LLM学習ソースになる | ハッシュタグとサービス名表記を統一（`13` §5） |
| コンテンツ→リファラル | 記事流入ユーザーは悩みが明確 → 相性占いへの誘導CVRが高い | 恋愛記事のCTAは相性占いに直結させる |
| リファラル→リテンション | 友人と一緒に使うユーザーはD30継続率が単独比+40%（仮説、`15`で検証） | 招待成立ペアには「ペア限定コンテンツ」を配信 |
| リテンション→リファラル | ストリーク報酬に「友達招待でストリーク保護」を組込む | `08_Viral_Strategy.md` §3参照 |

**判定文**: 各四半期レビューで、ループ間経路のうち相関係数が計測できていない経路が1つでもあれば、翌四半期の計測基盤整備を最優先バックログに積む。

### 1.4 フェーズ別の主力ループ

| フェーズ | 期間 | 主力ループ | 理由 | 副次ループ |
|---|---|---|---|---|
| Phase 0: 種まき | 0-6ヶ月 | リテンション + UGC | 少数ユーザーの熱量を最大化し「シェアされる体験」を完成させる | コンテンツ（仕込み） |
| Phase 1: 点火 | 6-18ヶ月 | UGC + コンテンツ | SEO資産が効き始め、UGCが臨界点を超える | リファラル（機能拡充） |
| Phase 2: 拡大 | 18-36ヶ月 | コンテンツ + リファラル | 検索・AI検索の複利とK-factor 0.45への引き上げ | 全ループ稼働 |
| Phase 3: ドミナント | 36ヶ月+ | 4ループ全稼働 + 多言語展開 | 国内関心層1/3到達に向けた面の制圧、海外へ横展開 | — |

### 2.1 オーガニック検索（SEO / AEO / LLMO）

> 詳細戦略・実行計画は [13_SEO_AEO_LLMO.md](./13_SEO_AEO_LLMO.md) を正本とする。ここでは要約のみ。

- **役割**: 中長期の最大流入源。「悩みが顕在化した瞬間」を捕捉する高CVRチャネル。コンテンツループの入口
- **ターゲット**: P1/P2/P4（悩み検索層）。AI検索経由ではP3/P5も
- **KPI（12ヶ月）**: オーガニック流入 月間100万セッション、記事→登録CVR 8%、AI検索言及率（主要4LLMで「おすすめAI占い」回答時の言及率）50%

- **成功判定**: 12ヶ月時点で新規獲得の25%以上をオーガニック検索（AI検索含む）が占めること

### 2.2 Instagram

- **役割**: ブランド世界観の中核メディア。「保存される」ことでアルゴリズム上の複利を得る。UGCループの主戦場（ストーリーズシェア）
- **ターゲットペルソナ**: P1（コア）、P4、P5
- **KPI**:

| 指標 | 3ヶ月 | 12ヶ月 |
|---|---|---|
| フォロワー | 10,000 | 150,000 |
| 保存率（保存数/リーチ） | 3% | 5% |
| プロフィール→リンククリック率 | 8% | 12% |
| 月間経由新規登録 | 1,500 | 30,000 |

- **投下リソース**: 専任1名 + デザイナー0.5名。月次制作: カルーセル12本、リール12本、ストーリーズ毎日
- **コンテンツ方針**:
  - カルーセル: 「保存して見返す」前提の悩み別ガイド（例:「復縁できる人の共通点7つ」）。最終ページに鑑定CTA
  - リール: 診断参加型・共感フック型（台本テンプレは `12` §4）
  - 世界観: 「夜空」トーン（Spine §9）でフィードを統一。恐怖訴求は絶対に使わない
- **初動90日プラン**:

| 週 | アクション |
|---|---|
| W1-2 | アカウント設計（プロフィール文・ハイライト・世界観ガイド確定）、カルーセル型テンプレ3種制作 |
| W3-6 | カルーセル週3本+リール週3本を定常運転。保存率をクリエイティブ別に計測 |
| W7-10 | 保存率上位20%の型に制作を寄せる。ストーリーズで鑑定結果シェアのリポスト運用開始 |
| W11-13 | フォロワー1万到達を判定。リンクスタンプ→登録CVRのA/Bを2本実施 |

- **成功判定基準**: 90日でフォロワー10,000未満 または 保存率2%未満なら、クリエイティブ戦略を全面改定（撤退ではなく改定。Instagramはブランド上の必須拠点）

### 2.3 TikTok

- **役割**: バイラル獲得の最大爆発力。診断系フォーマットで「やってみた」連鎖を作る。UGCループの点火装置
- **ターゲットペルソナ**: P5（Z世代）、P1
- **KPI**:

| 指標 | 3ヶ月 | 12ヶ月 |
|---|---|---|
| フォロワー | 20,000 | 300,000 |
| 完視聴率（15-30秒動画） | 35% | 45% |
| 10万再生超え動画 | 月1本 | 月6本 |
| 月間経由新規登録 | 2,000 | 50,000 |

- **投下リソース**: 専任1名（企画・出演・編集兼務）+ 月10万円の小道具/編集外注。投稿頻度: 毎日1本（90日で90本）
- **コンテンツ方針**: 「診断参加型」（コメント欄に生年月日→返信で一言鑑定）、「あるある型」（星座別あるある）、公開鑑定切り抜き。バズは量からしか生まれない — 90本投稿するまで型の断定をしない
- **初動90日プラン**:

| 週 | アクション |
|---|---|
| W1-2 | 競合・占いジャンルのバズ動画100本を分析、5型×各3本の初期ロット制作 |
| W3-8 | 毎日投稿。週次で完視聴率・コメント率をレビューし、翌週の型配分を変更 |
| W9-13 | 10万再生動画の再現実験（同型3連投）。コメント→鑑定誘導のオペレーション確立 |

- **成功判定基準**: 90日で「10万再生以上が累計3本未満」かつ「フォロワー1万未満」なら企画者を交代または外部クリエイター起用に切替。経由登録CVRはプロフィールリンク経由で計測し、登録単価換算¥500以下を維持

### 2.4 YouTube

- **役割**: ショートで認知、長尺で信頼。「悩み解説」長尺は検索資産（YouTube内検索+Google動画枠）として蓄積。指名検索の受け皿
- **ターゲットペルソナ**: P2、P4（長尺視聴耐性が高い）、P3
- **KPI**:

| 指標 | 6ヶ月 | 12ヶ月 |
|---|---|---|
| 登録者 | 10,000 | 50,000 |
| ショート: フォロワー転換率 | 0.5% | 1% |
| 長尺: 平均視聴維持率 | 35% | 45% |
| 月間経由新規登録 | 800 | 8,000 |

- **投下リソース**: 兼任1名（TikTok素材の転用が基本）+ 長尺は月4本を外部編集（1本¥30,000目安）
- **コンテンツ方針**: ショートはTikTok素材の横展開（週5本）。長尺は「復縁の心理学と占いの使い方」等、悩み×教養の20分解説（週1本）。概要欄に鑑定リンク+タイムスタンプ
- **初動90日プラン**: W1-4 ショート転用の運用ライン確立 → W5-8 長尺4本公開しCTR/維持率計測 → W9-13 維持率40%超の長尺フォーマットに集中
- **成功判定基準**: 6ヶ月で登録者1万未満なら長尺は月2本に減らしショート特化。ショートの経由登録単価が¥1,000を超え続ける場合はTikTok/リールへの素材集中に切替

### 2.5 X（旧Twitter）

- **役割**: 「毎日の運勢ポスト」で習慣接触を作る、リテンションループのSNS側出口。引用RT文化を利用したバズ設計とUGCの震源地
- **ターゲットペルソナ**: P1、P2、P3（テキスト親和層）
- **KPI**:

| 指標 | 3ヶ月 | 12ヶ月 |
|---|---|---|
| フォロワー | 15,000 | 200,000 |
| 毎日の運勢ポストの平均エンゲージ率 | 3% | 5% |
| 引用RT数（月間） | 500 | 10,000 |
| 月間経由新規登録 | 1,000 | 25,000 |

- **投下リソース**: 専任0.5名（CRM担当と兼務可）。毎朝7時の12星座運勢ポスト（自動生成→人間確認）+ 日中2投稿
- **バズ設計**: 「引用RTで自分の星座の結果を語りたくなる」構造（例: 12星座ランキング形式は自分の順位に言及したくなる）。バズ狙い投稿は週2本、`08_Viral_Strategy.md` §4のフック集を使用
- **初動90日プラン**: W1-2 毎朝運勢の自動生成パイプライン構築 → W3-8 ランキング型・診断型の週次実験 → W9-13 エンゲージ率上位フォーマットへ集中、インフルエンサーとの相互言及開始
- **成功判定基準**: 90日でフォロワー15,000未満なら投稿時刻・形式を全面改定。引用RT率がリプ率を下回り続ける場合は「語りたくなる余白」の設計をやり直す

### 2.6 LINE

- **役割**: **CRMの中核**（獲得チャネルというより維持・収益化チャネル）。友だち追加=最重要マイクロコンバージョン。全チャネルの「受け皿」であり、リテンションループの配信基盤
- **ターゲットペルソナ**: 全ペルソナ（日本のスマホユーザーの9割超が利用）
- **KPI**:

| 指標 | 3ヶ月 | 12ヶ月 |
|---|---|---|
| 友だち数 | 30,000 | 500,000 |
| ブロック率 | 8%以下 | 6%以下 |
| 朝運勢メッセージ開封率 | 45% | 50% |
| LINE経由の週次鑑定実施率 | 25% | 35% |

- **投下リソース**: 専任1名 + LINE公式アカウント費用（従量課金、月間配信数で変動。12ヶ月時点で月額200〜400万円を見込み予算化）
- **運用方針**: 全SNS・記事のCTAは「LINE友だち追加→その場で1回無料鑑定」に統一。配信は朝運勢（毎日）+ 週2のセグメント配信まで（配信過多はブロック率悪化）。詳細は [11_CRM.md](./11_CRM.md)
- **初動90日プラン**: W1-3 リッチメニュー・応答シナリオ・友だち追加時の初回鑑定フロー構築 → W4-8 朝運勢配信の開封率A/B（時刻・文面）→ W9-13 セグメント配信（悩みカテゴリ別）開始
- **成功判定基準**: ブロック率が単月10%を超えたら配信頻度を即時半減。友だち→サービス登録CVRが60%未満なら初回フローを改修

### 2.7 Pinterest

- **役割**: 「保存文化×検索型UI」の長寿命チャネル。1ピンが数年間流入を生む。カルーセル素材の転用先でありコンテンツループの補助入口
- **ターゲットペルソナ**: P1、P4（ビジュアル検索・ライフスタイル関心層）
- **KPI（12ヶ月）**: 月間インプレッション300万、アウトバウンドクリック月30,000、経由新規登録 月3,000
- **投下リソース**: 兼任0.2名。Instagram用カルーセル・記事アイキャッチを週10ピン転用（新規制作ゼロで運用）
- **初動90日プラン**: W1-2 ビジネスアカウント+ボード設計（恋愛/仕事/自己理解/今日の運勢）→ W3-13 週10ピンの定常転用、リッチピン設定
- **成功判定基準**: 転用のみ・追加コストほぼゼロのため撤退基準は緩く設定: 12ヶ月でアウトバウンドクリック月10,000未満なら頻度を週3ピンへ縮小

### 2.8 Google Discover

- **役割**: 「検索されない発見流入」。E-E-A-T・高品質画像・話題性を満たす記事がスマホホーム面に露出し、単発で数十万PVを生む
- **ターゲットペルソナ**: P1、P4（モバイルブラウジング層）
- **KPI（12ヶ月）**: Discover経由 月間30万セッション、Discover掲載記事比率 公開記事の10%
- **投下リソース**: 専任なし（SEOチームの運用要件に統合）。要件: 1200px以上のアイキャッチ、`max-image-preview:large`、話題性のあるタイトル（ただし釣りタイトル禁止 — ブランド毀損）

### 2.9 PR（広報）

- **役割**: 信頼の外部調達。「調査リリース」で報道価値を作り、テレビ・雑誌・Webメディア掲載を獲得。LLMO（AI検索の学習ソース化）の最重要供給源でもある（`13` §4）
- **ターゲットペルソナ**: 全体（特にP2/P4はマスメディア接触率が高い）
- **KPI**:

| 指標 | 6ヶ月 | 12ヶ月 |
|---|---|---|
| 調査リリース本数 | 4本 | 12本 |
| メディア掲載数（転載除く一次掲載） | 20件 | 100件 |
| テレビ・雑誌掲載 | 1件 | 6件 |
| 指名検索数（月間） | 5,000 | 50,000 |

- **投下リソース**: 兼任0.5名 + PR会社リテナー（月30万円）またはリリース配信サービス
- **コンテンツ方針**: 「恋愛と意思決定」の統計データ発信を軸にする。例:「20代女性の68%が『重要な決断の前に占いを見た経験がある』（n=1,000自社調査）」。自社鑑定データの匿名統計（「今月最も占われた悩みランキング」）は他社が真似できない独自資産
- **初動90日プラン**: W1-4 調査設計・第1回調査実施（n=1,000）→ W5-8 リリース配信+個別メディアピッチ30件 → W9-13 掲載メディアとの関係構築、第2回調査企画
- **成功判定基準**: リリース1本あたり一次掲載5件未満が2回続いたら、調査テーマ設計とピッチ先リストを全面見直し

### 2.10 インフルエンサー

- **役割**: 信頼の借用による初期認知の跳躍。UGCループの「種火」。広告ではなく「本人が使って語る」文脈を作る
- **ターゲットペルソナ**: 起用ジャンルで対応 — 占い系（P1/P4）、恋愛系（P1）、Vlog/ライフスタイル系（P2/P5）
- **KPI（12ヶ月）**: 稼働インフルエンサー100名、経由新規登録 月20,000、平均登録単価¥600以下
- **投下リソース**: 専任1名 + 月200万円（フェーズにより増額）
- **段階設計**:

| 段階 | 対象 | 報酬 | 判定 |
|---|---|---|---|
| 1. ギフティング | フォロワー1-10万のマイクロ層に無料Premium 3ヶ月付与 | 無償（PR表記必須） | 投稿率30%以上で次段階へ |
| 2. 成果報酬 | 投稿反応が良かった層 | 登録1件¥300-500（専用リンク計測） | 登録単価¥800以下を維持 |
| 3. アンバサダー | 継続的に成果を出す上位者 | 月額固定+成果報酬+共同企画 | §2.11へ移行 |

- **初動90日プラン**: W1-3 候補300名リストアップ（占い/恋愛/Vlog各100名）→ W4-8 ギフティング100名実施 → W9-13 投稿率・反応を評価し成果報酬契約20名締結
- **成功判定基準**: ギフティング投稿率20%未満なら「贈る体験パッケージ」（世界観のある招待キット）を再設計。成果報酬の登録単価¥1,000超が2ヶ月続いたらジャンル配分を変更
- **遵守事項**: ステマ規制（景表法）完全遵守。PR表記のない依頼は一切行わない（Spine §9の倫理と一体）

### 2.11 アンバサダー制度

- **役割**: 熱量上位1%のユーザーを組織化し、UGC・コミュニティ・口コミの持続的な発生源にする。「ファンが運営に参加する」構造でスイッチングコストを情緒面から構築
- **対象**: NPS 9-10かつ月間投稿1回以上のユーザーから招待制で選抜（MAU比0.1%→上限1%）
- **KPI（12ヶ月）**: アンバサダー500名、アンバサダー由来UGC 月2,000投稿、アンバサダーの平均紹介人数 月3人
- **提供価値（金銭より身分と関与）**:
  - 新機能の先行アクセスとフィードバック権
  - 限定バッジ・限定鑑定テーマ・年1回のオンライン感謝祭
  - 紹介成果に応じたチケット付与（`08` §5と共通の報酬系）
- **投下リソース**: コミュニティマネージャー0.5名（§2.15と兼務）
- **初動90日プラン**: W1-4 選抜基準・特典・行動規範の設計 → W5-8 初期50名を手動招待、専用グループ開設 → W9-13 アンバサダー発案の企画を1本公式実施
- **成功判定基準**: 6ヶ月時点でアンバサダーの月次アクティブ率70%未満なら特典体系を再設計。「報酬目当ての機械的投稿」が目立つ場合は選抜基準を厳格化

### 2.12 アフィリエイト（ASP）

- **役割**: 成果報酬型の獲得補完。占い・恋愛系メディア/ブロガーの在庫を刈り取る。CACが完全にコントロール可能な数少ないチャネル
- **KPI（12ヶ月）**: 提携メディア200件、経由新規登録 月10,000、承認率80%以上
- **単価設計**:

| 成果地点 | 単価 | 意図 |
|---|---|---|
| 無料登録 | ¥300 | 量の確保（不正リスク高、上限管理） |
| 初回課金 | ¥2,000 | 質の確保（主軸） |
| サブスク初月継続 | +¥1,000ボーナス | LTV連動へ誘導 |

- 初回課金成果¥2,000は、無料→有料転換率6%（Spine §5）・LTV/CAC>3の制約から逆算した上限内（サブスクLTV目安 ¥2,980÷解約率7%≒¥42,000 ≫ 3×CAC）

- **初動90日プラン**: W1-4 ASP選定・素材（バナー/LP）準備 → W5-8 上位占い系メディア30件へ個別提携打診 → W9-13 承認フロー・不正検知の運用確立
- **成功判定基準**: 経由ユーザーのD30継続率が全体平均の60%未満なら無料登録成果を停止し初回課金成果のみに絞る。**有料広告経由20%以下ルールの算入対象**（アフィリエイトは有料獲得としてカウントする）

### 2.13 企業提携（BD）

- **役割**: 「意思決定の後押し」という本質価値（Spine §1）が隣接する事業との相互送客。CACゼロ〜低額で質の高い流入を得る
- **提携先カテゴリと設計**:

| 提携先 | ユーザー文脈 | 提携形態 | 想定ペルソナ |
|---|---|---|---|
| マッチングアプリ | 「この人と合う？」の瞬間 | 相性占いのAPI提供/相互送客 | P1 |
| 美容（サロン・コスメEC） | 自分磨き・変わりたい | 特典として鑑定チケット配布 | P1/P4 |

### 2.14 ライブ配信（TikTok LIVE / Instagram LIVE）

- **役割**: 「公開鑑定」によるリアルタイムの信頼構築と、切り抜き素材の量産源。コメント参加が擬似体験となり登録CVRが高い

- **ターゲットペルソナ**: P1、P5（コミュニティ参加耐性が高い層）

## 3. リファラル/紹介プログラム設計（概要）

> 詳細設計・報酬テーブル・不正対策は [08_Viral_Strategy.md](./08_Viral_Strategy.md) を正本とする。

### 3.1 インセンティブ構造（双方向報酬）

| 対象 | 報酬 | タイミング |
|---|---|---|
| 招待した側 | 深掘り鑑定チケット1枚 | 被招待者が初回鑑定完了時 |
| 招待された側 | 通常より豪華な初回鑑定（＋チケット1枚） | 登録直後 |

- 現金・ポイントではなく**チケット（プロダクト内価値）**で付与する: CAC換算で安価、かつ従量課金（収益の25%、Spine §4）への導線になる
- 招待の主導線は「相性占い」（関係性占いは構造的に相手を必要とする — ループ3）。「紹介してください」ではなく「あの人との相性を見よう」と言う

### 3.2 K-factor目標と計算式

```
K = i × c
  i = 既存ユーザー1人が送る平均招待数（月次）
  c = 招待→登録CVR

12ヶ月目標: K = 0.25 = i 0.71 × c 0.35
36ヶ月目標: K = 0.45 = i 1.13 × c 0.40
```

- K<1でも複利効果は大きい: 自然流入N人はK=0.25で最終的に N×1/(1-0.25)=1.33倍 に増幅される（K=0.45なら1.82倍）
- **判定文**: iとcを分解計測し、cが0.35未満なら招待受け取り体験（LP・初回鑑定）を、iが目標未満なら招待トリガー（結果画面のシェア導線・相性占い露出）を改善する

## 4. 有料広告の位置づけ

### 4.1 原則

> **有料広告は「成長の主エンジン」ではなく「実験とデータ取得のための最小限の装置」である。**

- 有料広告経由（アフィリエイト含む）の新規獲得比率は**全社で20%以下**を毎月 `14_KPI_Dashboard.md` で監視。20%超過が2ヶ月続いたら広告出稿を自動縮小する（例外承認は事業責任者のみ）
- 広告の許容用途: ①クリエイティブ・メッセージの高速A/Bテスト ②新機能ローンチ時の初速データ取得 ③SEO/UGCが未成熟な初期のカンフル剤（Phase 0のみ）

### 4.2 出稿条件（すべて満たす場合のみ出稿可）

| 条件 | 基準 | 判定方法 |
|---|---|---|
| ユニットエコノミクス | 予測LTV/CAC > 3 | コホートLTVカーブから12ヶ月LTVを予測 |
| ペイバック | CAC回収 6ヶ月以内 | チャネル別コホートの累積粗利で判定 |
| クリエイティブ | UGC転用 or UGC風（広告らしい広告は作らない） | 実UGCの許諾転用を最優先（許諾フローは `12` §5） |
| 比率制約 | 出稿後も有料経由比率20%以下を維持できる出稿額 | 月次予算上限=当月予測新規×20%×目標CAC |
| 倫理 | 恐怖訴求・断定的不幸予言のクリエイティブ禁止 | 出稿前レビュー必須（Spine §9） |

### 4.3 媒体優先度

| 優先 | 媒体 | 用途 |
|---|---|---|
| 1 | Meta（Instagram/Facebook） | UGC転用クリエイティブのテスト・獲得 |
| 2 | TikTok Ads（Spark Ads） | バズったオーガニック投稿のブースト |
| 3 | Google検索（指名+高意図KW） | 指名検索の刈り取り防衛のみ |
| 4 | LINE広告 | 友だち追加広告（CRM基盤強化） |

## 6. グロースモデルの数式化

### 6.1 基本式

```
月間新規登録 A(t) = Σ_ch [ 流入(ch, t) × CVR(ch) ] × (1 + K(t)×増幅係数)

MAU(t) = MAU(t-1) × 継続率調整後残存率 + A(t)
       ≒ Σ_{k=0}^{t} A(t-k) × r(k)      ※r(k)=登録kヶ月後の月次残存率カーブ

課金者(t) = MAU(t) × 課金率(t)
月商(t)   = 課金者(t) × ARPPU（¥3,000）
```

- r(k)はコホート実測で毎月更新する。初期仮定: r(1)=25%, r(3)=18%, r(6)=15%, r(12)=13%（D30改善に伴い上方修正）
- 各チャネルのCVRは「流入→登録」で統一定義（`14_KPI_Dashboard.md` の定義正本に従う）

### 6.2 月商50億円へのマイルストーン（トップダウン整合）

| 時点 | MAU | 課金率 | 課金者 | 月商 |
|---|---|---|---|---|
| 12ヶ月 | 50万 | 6% | 3万 | 0.9億円 |
| 24ヶ月 | 250万 | 8% | 20万 | 6億円 |
| 36ヶ月 | 700万 | 9% | 63万 | 19億円 |
| 48ヶ月 | 1,200万 | 10% | 120万 | 36億円 |
| 60ヶ月 | 1,700万 | 10% | 167万 | **50億円** |

- 課金率はSpine §5の無料→有料転換率（12ヶ月6%→36ヶ月10%）の改善軌道と連動
- 36ヶ月以降は多言語展開（Spine §4）を含む。国内のみのMAU上限リスクは `17_Roadmap.md` で管理

### 6.4 感度分析（どのレバーが最も効くか）

| レバー+10%改善 | 60ヶ月時点の月商インパクト（概算） | 優先度 |
|---|---|---|
| D30継続率（15%→16.5%） | +8〜10%（残存カーブ全体が持ち上がる） | 最優先 |
| K-factor（0.45→0.50） | +9%（増幅係数1.82→2.0） | 高 |
| 無料→有料転換率（10%→11%） | +10%（線形） | 高 |
| 記事→登録CVR（8%→8.8%） | +2〜3% | 中 |
| 有料広告出稿額+10% | +0.3%（比率上限で頭打ち） | 低 |

- **結論**: 広告費の増額は最も効かないレバーである。継続率・K-factor・転換率への投資（=プロダクトとコンテンツ）を常に優先する（Spine §7の思想の定量的裏付け）

## 7. 計測とアトリビューション

### 7.1 計測の原則

- **チャネル別に「初回接触（ファーストタッチ）」と「登録直前（ラストタッチ）」を両方記録**し、§5のポートフォリオ判定はファーストタッチで、CAC計算はラストタッチで行う（二重計上を防ぎつつチャネルの役割差を可視化）
- 全SNSプロフィール・記事CTA・招待リンクは**チャネル別パラメータ付きURL**（utm_source/utm_medium/utm_campaign）で発行し、命名規約は `14_KPI_Dashboard.md` を正本とする
- リファラ欠損が大きいチャネル（Instagramストーリーズ・LINEトーク内シェア・口コミ）は、登録時アンケート「何で知りましたか？」（単一選択+自由記述）で補正する。アンケート回答率60%以上を維持する

### 7.2 チャネル共通の評価指標（全チャネルで同一定義）

| 指標 | 定義 | 用途 |
|---|---|---|
| 流入 | チャネル起点のセッション/プロフィールクリック | 上流の量 |
| 登録CVR | 流入→無料登録 | チャネルの質（意図の強さ） |
| WRS転換率 | 登録者のうち初週にWRS条件（鑑定完了+保存/シェア/メモ/再訪）到達した比率 | NSMへの寄与（Spine §3） |
| D7/D30継続率 | チャネル別コホートの継続 | 質の最終判定 |
| 実効CAC | チャネル直接費÷ラストタッチ登録数 | 予算配分 |
| 12ヶ月LTV予測 | コホートLTVカーブによる予測値 | LTV/CAC>3判定 |

- **判定文**: チャネル別D30継続率が全体平均の60%未満のチャネルは「量が出ていても」縮小候補とする。流入の量は質の代わりにならない（Spine §7: LTVを伸ばさない施策は実行しない）

### 7.3 週次チャネルレポートの標準フォーマット

| 項目 | 内容 |
|---|---|
| 1行サマリ | 「今週の最重要変化」を1文で（例: TikTok型2が完視聴率+8pt） |
| 数値表 | §7.2の6指標×全チャネル、前週比・目標比 |
| 実験結果 | 完了した実験のICE事後評価（`15_Experiment_Backlog.md` に反映） |
| 来週のアクション | チャネルごとに最大3つ、担当者名付き |

## 10. リスクと対応

| リスク | 兆候（数値トリガー) | 対応 |
|---|---|---|
| 単一SNSのアルゴリズム変更/凍結 | 特定チャネル流入が月次-40%超 | ポートフォリオ分散（§5）が保険。該当チャネル人員を他チャネルへ一時再配置。フォロワーのLINE移行を平時から実施（§2.6の受け皿設計） |
| UGCの炎上・ネガティブ拡散 | ネガティブ言及比率が週次10%超 | `09_Brand_Strategy.md` の危機対応プロトコル起動。恐怖訴求ゼロ方針（Spine §9）が予防の第一線 |
| ステマ規制・景表法違反 | — （予防のみ） | インフルエンサー契約書にPR表記義務・監査条項。違反投稿は即時取り下げ要請 |
| 占いカテゴリへのプラットフォーム規制強化 | 広告審査落ち・リーチ制限の頻発 | 「エンタメ+意思決定支援」のカテゴリポジション明確化、依存誘発表現の全面排除（元々の方針と一致） |
| K-factor目標未達 | 12ヶ月時点でK<0.15 | `08_Viral_Strategy.md` の招待導線を全面再設計。§6.4の感度分析が示す通り最重要レバーのため、プロダクトリソースを最優先投入 |
| CAC高騰（有料実験枠） | LTV/CAC<3が2ヶ月連続 | §4.2の条件により自動停止（裁量による継続は不可） |
