## レポートメタ情報ブロック

すべてのレポートは本文の前に以下のメタ情報ブロックを置く。**8項目すべて必須**。値が確定しない項目は「未定」ではなく、置いた前提を書く（SKILL.md Step 1 の完了判定に対応）。

```markdown
# {{結論を含むレポートタイトル}}

**{{サブタイトル: 対象と期間を1行で}}**

| 項目 | 内容 |
|---|---|
| 作成日 | {{YYYY-MM-DD}} |
| 調査期間 | {{YYYY-MM-DD}} 〜 {{YYYY-MM-DD}} |
| 作成者 | {{作成者名/チーム名}} |
| 対象読者 | {{経営層/投資家/実務者/社内共有 のいずれか}} |
| 機密区分 | {{社外秘/関係者限り/社内公開/公開可 のいずれか}} |
| 前提条件 | {{スコープ・省略章・仮置きした前提を列挙。なければ「特になし」}} |
```

判定基準:

- タイトルは結論を含む（型は `STYLE_GUIDE.md` §タイトル/サブタイトル設計）
- 「前提条件」には Step 1 で推測で置いたデフォルト、省略した章とその理由、データ基準日の注意をすべて書く（SKILL.md §3 エラーハンドリング準拠）
- 機密区分は必ず4択から選ぶ。判断できない場合は「社外秘」を既定とする

## 12章構成の全体像

| # | 章名 | 1行の目的 | 対応Step |
|---|---|---|---|
| 1 | Executive Summary | 全体を1ページに凝縮し、読者がこの章だけで意思決定できる状態にする | Step 10（最後に書く） |
| 2 | Purpose | このレポートが何を意思決定させるためのものかを確定する | Step 1 |
| 3 | Background | 読者と前提知識を揃え、なぜ今この調査が必要かを示す | Step 1 |
| 4 | Market Research | 市場の規模・成長性・構造を数値で示す | Step 2〜3 |
| 5 | Competitor Analysis | 競合の戦略・強み・弱みを比較し、自社の位置を特定する | Step 2〜4 |
| 6 | User Insights | ユーザーの行動・課題・ニーズから発見を抽出する | Step 2〜4 |
| 7 | Key Findings | 調査全体から意思決定を変えうる発見を3〜7個に絞る | Step 4〜5 |
| 8 | Strategic Recommendations | 戦略オプションを比較し、推奨案を1つに絞って提示する | Step 6〜7 |
| 9 | KPI | 目標をKPIツリーに構造化し、計測可能にする | Step 8 |
| 10 | Roadmap | 施策を時間軸に配置し、フェーズ移行の判定条件を置く | Step 9 |
| 11 | Risks | リスクを影響度×発生確率で評価し、対策を紐づける | Step 5〜6 |
| 12 | Next Actions | 誰が・何を・いつまでに、の粒度で直近の行動を確定する | Step 7、品質基準7 |

構成上のルール:

- 章の順序は入れ替えない。省略する場合は次節のマトリクスに従い、省略した章名と理由を「前提条件」に明記する
- 章番号と章名は `1. Executive Summary` のように「番号 + 半角ピリオド + 半角スペース + 英語章名」で表記する（`OUTPUT_RULES.md` §章番号・見出し規則と整合）
- 各章の第1文は結論（品質基準1）。書き出しの型は各章の定義に示す

## 各章の定義とスケルトン

### 1. Executive Summary

**目的（1行）**: 読者がこの章だけを読んで意思決定できる情報密度で、レポート全体を凝縮する。

**必須要素チェックリスト**（品質基準2に対応）:

- [ ] 結論（推奨する意思決定）が第1文にある
- [ ] 根拠となる数値が3個以上ある（市場規模・成長率・効果見込み等）
- [ ] 推奨アクションが1〜3個、優先度付きで書かれている
- [ ] 主要リスクが1個以上、対策とセットで書かれている
- [ ] 全体が1ページ以内（見出し込み40行以内、本文800文字以内）

**推奨分量**: 20〜40行 / 400〜800文字。表1つ + 箇条書きで構成する。

**図表の型**: 「結論サマリー表」（推奨案/期待効果/投資規模/主要リスクの4行表）。

**書き出しの型**: 「本レポートは {{結論}} を推奨する。」

**執筆順序の注意**: 本章は必ず**本文完成後に最後に書く**（SKILL.md Step 10）。本文にない数値・主張を本章に書いてはならない。

```markdown
## 1. Executive Summary

本レポートは {{推奨する意思決定を1文で}} を推奨する。

**結論**: {{結論を2〜3文で。数値を必ず含める}}

| 項目 | 内容 |
|---|---|
| 推奨案 | {{推奨案を1行で}} |
| 期待効果 | {{数値で。例: 3年目に売上◯億円}} |
| 投資規模 | {{数値または S/M/L}} |
| 主要リスク | {{最大のリスク1つ + 対策1行}} |

**推奨アクション（優先度順）**:

1. 🔴 {{最優先アクション}} — {{期限}}
2. 🟡 {{次点アクション}} — {{期限}}
3. 🟢 {{その次のアクション}} — {{期限}}
```

### 2. Purpose

**目的（1行）**: このレポートが「誰に・何を・意思決定させるためのものか」を1画面で確定する。

**必須要素チェックリスト**:

- [ ] 意思決定してほしいこと（Decision to make）が疑問文または選択肢の形で書かれている
- [ ] 対象読者と、読者がこのレポートをどう使うかが書かれている
- [ ] スコープ（対象市場・期間・地域）とスコープ外が明記されている

**推奨分量**: 10〜20行 / 200〜400文字。

**図表の型**: 「スコープ表」（対象/期間/地域/スコープ外の4行表）。

**書き出しの型**: 「本レポートの目的は {{意思決定事項}} の判断材料を提供することである。」

```markdown
## 2. Purpose

本レポートの目的は {{意思決定事項}} の判断材料を提供することである。

**意思決定事項**: {{「◯◯に参入すべきか」「A案とB案のどちらを選ぶか」等の疑問文}}

| スコープ | 内容 |
|---|---|
| 対象市場 | {{市場名}} |
| 期間 | {{分析対象期間}} |
| 地域 | {{対象地域}} |
| スコープ外 | {{今回扱わない範囲を明記}} |
```

### 3. Background

**目的（1行）**: 読者の前提知識を揃え、「なぜ今この調査が必要か」を事実ベースで示す。

**必須要素チェックリスト**:

- [ ] 調査に至った経緯（きっかけとなった事実・変化）が書かれている
- [ ] 現状の課題または機会が数値付きで1個以上示されている
- [ ] 既存の取り組み・過去の意思決定との関係が書かれている（初回調査なら「初回」と明記）

**推奨分量**: 15〜30行 / 300〜600文字。

**図表の型**: 必要に応じて「経緯タイムライン」（Mermaid timeline、ノード6個以内）。

**書き出しの型**: 「{{きっかけとなった事実}} を受け、{{調査対象}} の検討が必要になった。」

```markdown
## 3. Background

{{きっかけとなった事実}} を受け、{{調査対象}} の検討が必要になった。

**現状の課題/機会**:

- {{課題または機会1: 数値を含める}}
- {{課題または機会2: 数値を含める}}

**これまでの経緯**: {{過去の取り組み・意思決定との関係。初回なら「本テーマの調査は今回が初回である」}}
```

### 4. Market Research

**目的（1行）**: 市場の規模・成長性・構造を数値で示し、機会の大きさを判定可能にする（品質基準6）。

**必須要素チェックリスト**:

- [ ] 市場規模（TAM/SAM/SOM または総市場規模）が数値と出典付きで書かれている
- [ ] 成長率（CAGR等）が数値と出典付きで書かれている
- [ ] 市場の構造（セグメント・バリューチェーン・主要プレイヤー分布のいずれか）が図表で示されている
- [ ] 主要トレンドが2〜4個、事実ベースで書かれている
- [ ] すべての外部数値に脚注形式の出典が付いている（品質基準8、`OUTPUT_RULES.md` §脚注）

**推奨分量**: 40〜80行 / 800〜1,600文字。図表2〜4個。

**図表の型**: 「市場規模表」（年度×規模の推移表）、「セグメント構成比表」、必要なら Mermaid pie。

**書き出しの型**: 「{{市場名}} の市場規模は {{数値}}（{{基準年}}）であり、{{結論的評価}}。」

```markdown
## 4. Market Research

{{市場名}} の市場規模は {{N億円}}（{{YYYY年}}時点）であり、{{「年率N%で成長している」等の結論}}[^1]。

### 4.1 市場規模と成長性

| 年 | 市場規模 | 前年比 |
|---|---:|---:|
| {{YYYY}} | {{N億円}} | — |
| {{YYYY}} | {{N億円}} | {{+N%}} |
| {{YYYY}}（予測） | {{N億円}} | {{+N%}} |

### 4.2 市場構造

{{セグメント構成・プレイヤー分布を表またはMermaidで}}

### 4.3 主要トレンド

1. **{{トレンド1}}** — {{事実 + 出典}}
2. **{{トレンド2}}** — {{事実 + 出典}}
```

### 5. Competitor Analysis

**目的（1行）**: 競合の戦略・強み・弱みを同一軸で比較し、自社の勝ち筋（ポジション）を特定する。

**必須要素チェックリスト**:

- [ ] 競合が3〜5社選定され、選定理由が1行で書かれている
- [ ] 全競合を同一の比較軸（5軸以内）で並べた比較表がある
- [ ] 各競合の強み・弱みが事実（数値・実績）に基づいて書かれている
- [ ] 「自社が取るべきポジション」の示唆が最終節にある
- [ ] ポジショニングを示す場合は2軸で図示されている（Mermaid quadrantChart または表）

**推奨分量**: 40〜80行 / 800〜1,600文字。比較表1個 + ポジショニング図1個。

**図表の型**: 「競合比較表」（競合5社以内 × 比較軸5列以内）、「ポジショニングマップ」（quadrantChart、ノード12個以内）。

**書き出しの型**: 「競合{{N}}社の比較から、{{自社の勝ち筋に関する結論}}。」

```markdown
## 5. Competitor Analysis

競合{{N}}社の比較から、{{「◯◯領域が空白であり自社の参入余地がある」等の結論}}。

**選定理由**: {{選定基準を1行で}}

### 5.1 競合比較

| 比較軸 | {{A社}} | {{B社}} | {{C社}} |
|---|---|---|---|
| {{価格帯}} | {{値}} | {{値}} | {{値}} |
| {{主要機能}} | {{値}} | {{値}} | {{値}} |
| {{市場シェア}} | {{N%}}[^2] | {{N%}} | {{N%}} |

### 5.2 ポジショニング

{{quadrantChart または 2軸表}}

### 5.3 自社への示唆

- {{示唆1: 空白ポジション・差別化余地}}
- {{示唆2}}
```

### 6. User Insights

**目的（1行）**: ユーザーの行動・課題・ニーズの事実から、意思決定を変えうる発見を抽出する。

**必須要素チェックリスト**:

- [ ] 対象ユーザーセグメントの定義が書かれている
- [ ] 行動・課題の事実が出典（調査手法・データソース）付きで3個以上ある
- [ ] 「事実 → 解釈 → So What」の3段が揃ったインサイトが1個以上ある（Step 4の定義準拠）
- [ ] 定性情報（声・行動観察）と定量情報（数値）が区別されている

**推奨分量**: 30〜60行 / 600〜1,200文字。

**図表の型**: 「セグメント×課題表」、必要なら「カスタマージャーニー簡易表」（フェーズ5列以内）。

**書き出しの型**: 「ユーザー調査から、{{最重要インサイトの結論}}。」

```markdown
## 6. User Insights

ユーザー調査から、{{最重要インサイトを1文で}}。

### 6.1 対象セグメント

| セグメント | 定義 | 規模 |
|---|---|---:|
| {{S1}} | {{定義}} | {{N万人}} |

### 6.2 主要な事実

- **事実**: {{観察された行動・数値}}[^3]
- **事実**: {{観察された行動・数値}}

### 6.3 インサイト

| 事実 | 解釈 | So What |
|---|---|---|
| {{事実}} | {{なぜそうなるか}} | {{意思決定への含意}} |
```

### 7. Key Findings

**目的（1行）**: 調査全体（4〜6章）から、意思決定を変えうる発見だけを3〜7個に絞って提示する。

**必須要素チェックリスト**:

- [ ] Findingsが3〜7個である（Step 4完了判定と同一。8個以上なら統合、2個以下なら調査に戻る）
- [ ] 各Findingが「事実 → 解釈 → So What」を1セットで含む
- [ ] 各Findingに根拠章（4〜6章のどこに基づくか）への参照がある
- [ ] 事実の羅列（So Whatのないもの）が混入していない

**推奨分量**: 20〜50行 / 500〜1,000文字。Finding 1個につき3〜5行。

**図表の型**: 「Findings一覧表」または番号付きリスト。重要度を🔴🟡🟢で1系統表示可。

**書き出しの型**: 「調査全体から、意思決定に影響する発見は次の{{N}}点である。」

```markdown
## 7. Key Findings

調査全体から、意思決定に影響する発見は次の{{N}}点である。

| # | 重要度 | Finding（結論） | 根拠 |
|---|---|---|---|
| F1 | 🔴 | {{So Whatを含む1文}} | §4.1 |
| F2 | 🟡 | {{So Whatを含む1文}} | §5.2 |
| F3 | 🟢 | {{So Whatを含む1文}} | §6.3 |

### F1. {{Finding 1のタイトル}}

- **事実**: {{数値・出典付きの事実}}
- **解釈**: {{事実が意味すること}}
- **So What**: {{意思決定にどう効くか}}
```

### 8. Strategic Recommendations

**目的（1行）**: 戦略オプションを比較したうえで推奨案を1つに絞り、実行可能な施策に分解して提示する。

**必須要素チェックリスト**（Step 6〜7完了判定と対応）:

- [ ] 戦略オプションが2〜3案あり、評価軸（インパクト/実現性/リスク/投資額）の比較表がある
- [ ] 推奨案が1つに確定し、推奨理由が書かれている（推奨しない案も表に残す）
- [ ] 各推奨施策が「推奨内容/根拠/期待効果/リスク/優先度」の5点セットで書かれている（`OUTPUT_RULES.md` §推奨事項の書式）
- [ ] 施策が5個以上あり、全件にICEまたはRICEスコアと優先順位が付いている
- [ ] 根拠が Key Findings（F1〜）への参照になっている

**推奨分量**: 50〜100行 / 1,000〜2,000文字。本レポートの中核章であり、12章中最大の分量を許容する。

**図表の型**: 「オプション比較表」（3案×4評価軸）、「施策優先順位表」（ICE/RICEスコア付き）。

**書き出しの型**: 「{{N}}案を比較した結果、{{推奨案名}} を推奨する。」

### 9. KPI

**目的（1行）**: 目標をKPIツリーに構造化し、全指標を計測可能な状態で定義する。

**必須要素チェックリスト**（Step 8完了判定と対応）:

- [ ] KPIツリーが1枚の図（Mermaid flowchart または表）で表現されている
- [ ] 各KPIに「定義式/現状値/目標値/計測方法/確認頻度」の5項目がある
- [ ] 現状値が不明なKPIは「要計測」と明記されている（空欄・捏造禁止）
- [ ] 最上位目標が施策（8章）と接続している

**推奨分量**: 25〜50行。KPIツリー図1個 + KPI定義表1個。

**図表の型**: 「KPIツリー」（Mermaid flowchart TD、ノード12個以内。超える場合は表に落とす）、「KPI定義表」。

**書き出しの型**: 「最上位目標を {{目標}} とし、以下のKPIツリーで計測する。」

### 10. Roadmap

**目的（1行）**: 全施策を時間軸に配置し、フェーズ移行のGo/No-Go判定条件を数値で置く。

**必須要素チェックリスト**（Step 9完了判定と対応）:

- [ ] 8章の全施策がロードマップ上に配置されている
- [ ] フェーズ（または四半期）ごとにマイルストーンがある
- [ ] フェーズ移行の判定条件（Go/No-Go基準）が数値で書かれている
- [ ] 施策間の依存関係が明示されている（なければ「依存なし」と書く）

**推奨分量**: 25〜50行。ガント図または表1個 + 判定条件表1個。

**図表の型**: 「フェーズ表」または Mermaid gantt（タスク12個以内。超える場合はフェーズ表に落とす）。
